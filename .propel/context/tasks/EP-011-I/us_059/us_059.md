# User Story - US_059

## Story ID

* ID: US_059

## Story Title

* WCAG 2.2 AA & Semantic HTML

## Description

* As a Patient with accessibility needs, I want the application to meet WCAG 2.2 AA standards with proper semantic HTML, so that I can use assistive technologies (screen readers, magnifiers) to complete all tasks.

## Acceptance Criteria

1. **Given** semantic HTML (UXR-201), **When** any page renders, **Then** content uses proper landmark roles (header, nav, main, footer), heading hierarchy (h1-h6 without skipping levels), and ARIA labels on interactive elements.

2. **Given** color contrast (UXR-204), **When** text is displayed, **Then** normal text achieves ≥ 4.5:1 contrast ratio and large text achieves ≥ 3:1 against its background, validated by automated axe-core tests.

3. **Given** screen reader compatibility (UXR-206), **When** a screen reader traverses the page, **Then** all interactive elements have accessible names, dynamic content updates announce via ARIA live regions, and decorative images have empty alt attributes.

4. **Given** focus visibility, **When** a user navigates via keyboard, **Then** all focusable elements display a visible focus indicator that meets the 2px minimum outline requirement.

## Edge Cases

* What happens when dynamic content loads (e.g., search results)? An ARIA live region with role="status" announces the update count without interrupting the current reading flow.
* How are complex data tables handled? Tables include proper `<th>` scope attributes, `<caption>` elements, and `aria-describedby` for supplementary notes.
