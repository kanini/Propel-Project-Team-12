# User Story - US_002

## Story ID

* ID: US_002

## Story Title

* Backend Project Scaffolding with .NET 8

## Description

* As a developer, I want a properly structured .NET 8 ASP.NET Core Web API project following three-layer architecture, so that backend development follows clean separation of concerns from the start.

## Acceptance Criteria

1. **Given** the repository is ready, **When** the backend project is scaffolded, **Then** a solution file `PatientAccess.sln` exists with three projects: `PatientAccess.Web`, `PatientAccess.Business`, and `PatientAccess.Data`.

2. **Given** the three-layer architecture is implemented, **When** I inspect project references, **Then** `PatientAccess.Web` references `PatientAccess.Business`, `PatientAccess.Business` references `PatientAccess.Data`, and `PatientAccess.Data` has no reference to `PatientAccess.Web`.

3. **Given** the backend project is scaffolded, **When** I run `dotnet build`, **Then** the solution compiles without errors targeting .NET 8.0.

4. **Given** dependency injection is configured, **When** I inspect `Program.cs`, **Then** services, repositories, and middleware are registered following the DI pattern with proper interface-to-implementation mappings.

5. **Given** the project structure follows design.md, **When** I inspect the folder layout, **Then** Controllers, Services, Repositories, Models, DTOs, and Middleware folders exist in their respective layers.

## Edge Cases

* What happens when .NET SDK version is not 8.x? The build should fail with a global.json-enforced version requirement.
* How does the system handle missing NuGet packages on first build? `dotnet restore` should execute automatically and resolve all dependencies.
