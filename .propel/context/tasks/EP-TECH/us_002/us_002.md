# User Story - US_002

## Story ID
* ID: US_002

## Story Title
* Backend API Foundation (.NET 8 Minimal APIs)

## Description
* As a backend developer, I want a bootstrapped .NET 8 ASP.NET Core Web API project with dependency injection and minimal API structure, so that I can build REST endpoints efficiently following modern .NET patterns.

## Acceptance Criteria
1. **Given** I navigate to the `/api` folder, **When** I run `dotnet --version`, **Then** the project targets .NET 8.0 SDK.

2. **Given** the .NET 8 project is created, **When** I inspect the project structure, **Then** I find:
   - `Program.cs` with minimal API configuration and dependency injection container setup
   - `.csproj` file with `<TargetFramework>net8.0</TargetFramework>` and necessary NuGet packages
   - Layered folder structure: `/Domain`, `/Application`, `/Infrastructure`, `/API`
   - `appsettings.json` and `appsettings.Development.json` for configuration management

3. **Given** the API project is configured, **When** I run `dotnet run` in the `/api` folder, **Then** the application starts on `https://localhost:5001` without errors.

4. **Given** the API is running, **When** I navigate to `https://localhost:5001`, **Then** I receive a valid HTTP 200 or 404 response (no 500 errors), confirming the server is operational.

## Edge Cases
* What happens when the developer doesn't have .NET 8 SDK installed? (README includes prerequisite check commands)
* How does the project handle HTTPS certificate trust issues during local development? (`dotnet dev-certs https --trust` documented in setup guide)

## Technical Notes
- Use `dotnet new webapi` as starting point, then refactor to minimal APIs
- Install essential NuGet packages:
  - `Microsoft.AspNetCore.OpenApi`
  - `Microsoft.EntityFrameworkCore.Design`
- Configure dependency injection in `Program.cs` using `WebApplicationBuilder.Services`
- Folder structure aligns with Clean Architecture:
  ```
  /api
    /Domain          # Entities, value objects, domain events
    /Application     # Business logic, interfaces
    /Infrastructure  # External dependencies (DB, APIs)
    /API             # Controllers, Program.cs
  ```
