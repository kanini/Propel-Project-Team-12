# User Story - US_002

## Story ID
   * ID Format: US_002

## Story Title
   * Backend API Project Setup with .NET 8

## Description
  * As a developer, I want to scaffold the .NET 8 ASP.NET Core Web API project with three-layer architecture (Web, Business, Data), so that I have a maintainable backend structure following enterprise patterns that supports HIPAA compliance and clear separation of concerns.

## Acceptance Criteria
  * **Given** I have .NET 8 SDK installed, **When** I run the project creation scripts, **Then** three projects are created (PatientAccess.Web, PatientAccess.Business, PatientAccess.Data) with proper references and solution file.

  * **Given** the projects are created, **When** I inspect the directory structure, **Then** I see Controllers, Middleware, Services, Repositories, Models, and Configurations organized according to design.md architecture.

  * **Given** the Web API project is configured, **When** I run `dotnet run`, **Then** the API starts on localhost:5000 (HTTP) and localhost:5001 (HTTPS) with Swagger UI accessible at /swagger endpoint.

  * **Given** dependency injection is configured, **When** I request a service from the container, **Then** all dependencies are resolved correctly with proper lifetimes (Scoped for repositories, Singleton for caching).

  * **Given** the solution is configured, **When** I run `dotnet build`, **Then** all three projects compile successfully with no warnings in Release mode.

  * **Given** logging is configured with Serilog, **When** the application starts and processes requests, **Then** structured logs are written to console and configured sinks with appropriate log levels.

## Edge Cases
   * What happens when .NET SDK version is below 8.0? (Display error with required SDK version and download link)
   * How does the project handle missing configuration sections? (Validate required configuration at startup and fail fast with clear messages)
   * What if HTTPS certificate is not trusted? (Provide instructions to trust development certificate using dotnet dev-certs)
