# User Story - US_007

## Story ID
* ID: US_007

## Story Title
* Frontend Testing Framework Setup

## Description
* As a frontend developer, I want Jest and React Testing Library configured in the Next.js project, so that I can write unit and integration tests for React components with React-specific testing utilities.

## Acceptance Criteria
1. **Given** the `/client` Next.js project exists, **When** I inspect `package.json`, **Then** I find dev dependencies:
   - `jest` (version 29.x)
   - `@testing-library/react` (version 14.x)
   - `@testing-library/jest-dom` (version 6.x)
   - `@testing-library/user-event` (version 14.x)
   - `jest-environment-jsdom` for DOM testing

2. **Given** Jest is installed, **When** I inspect the project root, **Then** I find:
   - `jest.config.js` or `jest.config.ts` with Next.js-compatible configuration
   - `jest.setup.js` importing `@testing-library/jest-dom/extend-expect`
   - `package.json` includes test script: `"test": "jest --watch"`

3. **Given** the testing framework is configured, **When** I run `npm test` from the `/client` folder, **Then** Jest starts in watch mode and displays "No tests found" message (initial state).

4. **Given** Jest and React Testing Library are set up, **When** I create a sample component test:
   ```tsx
   import { render, screen } from '@testing-library/react';
   import Home from '@/app/page';

   describe('Home', () => {
     it('renders welcome message', () => {
       render(<Home />);
       const heading = screen.getByRole('heading');
       expect(heading).toBeInTheDocument();
     });
   });
   ```
   **Then** the test passes when executed with `npm test`.

## Edge Cases
* What happens when TypeScript types are missing for Jest matchers? (Install `@types/jest` to provide type definitions)
* How does Jest handle Next.js specific imports (e.g., `next/image`, `next/link`)? (Configure Jest to mock Next.js modules using `moduleNameMapper` in `jest.config.js`)

## Technical Notes
- Install testing packages:
  ```bash
  npm install --save-dev jest @testing-library/react @testing-library/jest-dom @testing-library/user-event jest-environment-jsdom
  ```
- `jest.config.js` template for Next.js:
  ```javascript
  const nextJest = require('next/jest')

  const createJestConfig = nextJest({
    dir: './',
  })

  const customJestConfig = {
    setupFilesAfterEnv: ['<rootDir>/jest.setup.js'],
    testEnvironment: 'jest-environment-jsdom',
    moduleNameMapper: {
      '^@/(.*)$': '<rootDir>/$1',
    },
  }

  module.exports = createJestConfig(customJestConfig)
  ```
- `jest.setup.js`:
  ```javascript
  import '@testing-library/jest-dom/extend-expect'
  ```
- Update `package.json` scripts:
  ```json
  {
    "scripts": {
      "test": "jest --watch",
      "test:ci": "jest --ci"
    }
  }
  ```
- Create `__tests__` folder for test files or co-locate tests with components
