# User Story - US_005

## Story ID
* ID: US_005

## Story Title
* Local Development Environment (Docker Compose)

## Description
* As a developer, I want a Docker Compose configuration that provisions PostgreSQL with pgvector extension and Redis containers, so that I can run the full application stack locally without manual database and cache server installation.

## Acceptance Criteria
1. **Given** Docker Desktop is installed, **When** I run `docker compose up -d` from the project root, **Then** two containers start successfully:
   - PostgreSQL 16+ container with pgvector extension enabled
   - Redis 7+ container for caching

2. **Given** the Docker Compose stack is running, **When** I run `docker compose ps`, **Then** both containers show status "Up" and expose ports:
   - PostgreSQL on `localhost:5432`
   - Redis on `localhost:6379`

3. **Given** PostgreSQL container is running, **When** I connect using `psql` or a database client (DataGrip, DBeaver), **Then** I can query `SELECT * FROM pg_extension WHERE extname = 'vector';` and see the pgvector extension installed.

4. **Given** the local environment is documented, **When** I read the README or `SETUP.md`, **Then** I find:
   - Prerequisites (Docker Desktop installation)
   - `docker compose up -d` command to start services
   - `docker compose down` command to stop services
   - Connection string examples for PostgreSQL and Redis
   - Troubleshooting tips for common Docker issues

## Edge Cases
* What happens when ports 5432 or 6379 are already in use by other services? (Document port conflict resolution and provide alternative port configuration in `docker-compose.yml`)
* How does the system handle Docker volume persistence for database data? (Configure named volumes in `docker-compose.yml` to persist data across container restarts)

## Technical Notes
- `docker-compose.yml` template:
  ```yaml
  version: '3.8'
  
  services:
    postgres:
      image: pgvector/pgvector:pg16
      container_name: patient-access-db
      environment:
        POSTGRES_USER: postgres
        POSTGRES_PASSWORD: yourpassword
        POSTGRES_DB: PatientAccessDb
      ports:
        - "5432:5432"
      volumes:
        - postgres_data:/var/lib/postgresql/data
      healthcheck:
        test: ["CMD-SHELL", "pg_isready -U postgres"]
        interval: 10s
        timeout: 5s
        retries: 5

    redis:
      image: redis:7-alpine
      container_name: patient-access-redis
      ports:
        - "6379:6379"
      volumes:
        - redis_data:/data
      healthcheck:
        test: ["CMD", "redis-cli", "ping"]
        interval: 10s
        timeout: 5s
        retries: 5

  volumes:
    postgres_data:
    redis_data:
  ```
- Use `pgvector/pgvector:pg16` official image to avoid manual extension installation
- Document Redis connection string: `localhost:6379` (no password for local development)
- Add `.env.example` with example environment variables
