# User Story - US_010

## Story ID
* ID: US_010

## Story Title
* Structured Logging Infrastructure

## Description
* As a backend developer, I want Serilog configured with Application Insights sink and correlation ID middleware, so that all API requests are logged with structured, searchable data and correlated across microservices for debugging and monitoring.

## Acceptance Criteria
1. **Given** the .NET 8 API project exists, **When** I inspect the `.csproj` file, **Then** I find NuGet packages:
   - `Serilog.AspNetCore` (version 8.x)
   - `Serilog.Sinks.ApplicationInsights` (version 4.x) OR `Serilog.Sinks.Console` for local development
   - `Serilog.Enrichers.CorrelationId`

2. **Given** Serilog is configured, **When** I review `Program.cs`, **Then** I find:
   - Serilog logger configured as the default logging provider
   - Log output template includes timestamp, correlation ID, log level, message, and exception details
   - Minimum log level set to `Information` for production, `Debug` for development

3. **Given** correlation ID middleware is implemented, **When** an HTTP request is received, **Then** the system:
   - Generates a unique correlation ID (GUID) if not present in `X-Correlation-ID` request header
   - Adds correlation ID to response headers
   - Enriches all log entries within the request scope with the correlation ID

4. **Given** the logging infrastructure is operational, **When** I make an API request to any endpoint, **Then** Serilog writes a structured log entry to the configured sink (console in development, Application Insights in production) containing:
   - Timestamp
   - Correlation ID
   - Log level (Information, Warning, Error)
   - HTTP method and path
   - Response status code
   - Request duration in milliseconds

## Edge Cases
* What happens when Application Insights connection string is missing or invalid? (Fallback to console logging and log warning about missing configuration)
* How does the system handle log flooding from high-traffic endpoints? (Configure sampling for Application Insights to reduce ingestion costs)

## Technical Notes
- Install Serilog packages:
  ```bash
  dotnet add package Serilog.AspNetCore
  dotnet add package Serilog.Sinks.Console
  dotnet add package Serilog.Sinks.ApplicationInsights
  dotnet add package Serilog.Enrichers.CorrelationId
  ```
- Configure Serilog in `Program.cs`:
  ```csharp
  using Serilog;
  using Serilog.Events;

  var builder = WebApplication.CreateBuilder(args);

  // Configure Serilog
  Log.Logger = new LoggerConfiguration()
      .MinimumLevel.Information()
      .MinimumLevel.Override("Microsoft", LogEventLevel.Warning)
      .MinimumLevel.Override("System", LogEventLevel.Warning)
      .Enrich.FromLogContext()
      .Enrich.WithCorrelationId()
      .WriteTo.Console(outputTemplate: "[{Timestamp:HH:mm:ss} {Level:u3}] {CorrelationId} {Message:lj}{NewLine}{Exception}")
      .WriteTo.ApplicationInsights(
          builder.Configuration["ApplicationInsights:InstrumentationKey"],
          TelemetryConverter.Traces)
      .CreateLogger();

  builder.Host.UseSerilog();

  var app = builder.Build();

  // Correlation ID middleware
  app.Use(async (context, next) =>
  {
      var correlationId = context.Request.Headers["X-Correlation-ID"].FirstOrDefault() 
          ?? Guid.NewGuid().ToString();
      context.Response.Headers.Add("X-Correlation-ID", correlationId);
      
      using (Serilog.Context.LogContext.PushProperty("CorrelationId", correlationId))
      {
          await next();
      }
  });

  app.Run();
  ```
- Add Application Insights instrumentation key to `appsettings.json`:
  ```json
  {
    "ApplicationInsights": {
      "InstrumentationKey": "your-instrumentation-key"
    }
  }
  ```
- For local development, use console sink; for production, use Application Insights sink
- Document correlation ID usage in API documentation for client integration
