# User Story - US_010

## Story ID
   * ID Format: US_010

## Story Title
   * Windows Services and IIS Deployment Support

## Description
  * As a developer, I want to configure the .NET backend for deployment as a Windows Service and IIS-hosted application, so that healthcare organizations can deploy on-premise infrastructure meeting their compliance and security requirements.

## Acceptance Criteria
  * **Given** the backend uses Microsoft.Extensions.Hosting.WindowsServices package, **When** I publish the application with Windows Service configuration, **Then** the executable can be registered and run as a Windows Service using sc.exe commands.

  * **Given** the Windows Service is registered, **When** the service starts, **Then** the ASP.NET Core application runs in the background, logs to Windows Event Log, and automatically restarts on failure.

  * **Given** IIS hosting is configured, **When** I publish the application with web.config transform, **Then** an ASP.NET Core Module handler is configured for IIS with correct process model (InProcess vs OutOfProcess).

  * **Given** the application is deployed to IIS, **When** IIS Application Pool runs the app, **Then** environment variables, connection strings, and appsettings are loaded correctly from IIS configuration.

  * **Given** HTTPS binding is configured in IIS, **When** requests are made to the API, **Then** SSL certificate is validated correctly and connections use TLS 1.2+ per NFR-004.

  * **Given** deployment documentation exists, **When** an administrator follows the guide, **Then** they can successfully deploy to both Windows Service and IIS without prior .NET deployment experience.

## Edge Cases
   * What happens when Windows Service fails to start due to permission issues? (Log detailed error to Event Log with required permissions checklist)
   * How does IIS deployment handle application pool recycling? (Implement graceful shutdown with IHostApplicationLifetime; complete in-flight requests before stop)
   * What if web.config is missing or misconfigured in IIS? (ASP.NET Core Module returns 500.19 error with specific configuration issue details)
