# User Story - US_008

## Story ID
* ID: US_008

## Story Title
* CI/CD Pipeline - Build & Test Workflows

## Description
* As a DevOps engineer, I want GitHub Actions workflows that automatically build and test both backend and frontend code on every push, with test coverage reporting and ≥80% coverage gate, so that code quality is maintained through automated checks.

## Acceptance Criteria
1. **Given** the repository is hosted on GitHub, **When** I inspect `.github/workflows/`, **Then** I find:
   - `backend-ci.yml` workflow for .NET 8 API build and test
   - `frontend-ci.yml` workflow for Next.js app build and test

2. **Given** the backend workflow is configured, **When** code is pushed to any branch, **Then** GitHub Actions executes:
   - `dotnet restore` to restore NuGet packages
   - `dotnet build --configuration Release` to compile the API
   - `dotnet test --collect:"XPlat Code Coverage"` to run xUnit tests with coverage
   - Workflow fails if build errors occur or tests fail

3. **Given** the frontend workflow is configured, **When** code is pushed to any branch, **Then** GitHub Actions executes:
   - `npm ci` to install dependencies (clean install)
   - `npm run build` to create production build
   - `npm run test:ci` to run Jest tests with coverage
   - Workflow fails if build errors occur or tests fail

4. **Given** test coverage is collected, **When** I review the workflow run summary, **Then** I see:
   - Backend coverage report showing percentage of lines/branches covered
   - Frontend coverage report showing percentage of statements/functions/lines/branches covered
   - ≥80% coverage threshold enforced (workflow fails if coverage below threshold)

## Edge Cases
* What happens when workflows run concurrently on multiple branches? (GitHub Actions handles parallelism; each branch gets isolated runner)
* How does the system handle flaky tests that intermittently fail? (Document retry strategy and test stability requirements in CONTRIBUTING.md)

## Technical Notes
- `.github/workflows/backend-ci.yml` template:
  ```yaml
  name: Backend CI

  on:
    push:
      branches: ['**']
    pull_request:
      branches: [main, develop]

  jobs:
    build-and-test:
      runs-on: ubuntu-latest

      steps:
        - uses: actions/checkout@v4
        
        - name: Setup .NET
          uses: actions/setup-dotnet@v4
          with:
            dotnet-version: 8.0.x
        
        - name: Restore dependencies
          run: dotnet restore
          working-directory: ./api
        
        - name: Build
          run: dotnet build --configuration Release --no-restore
          working-directory: ./api
        
        - name: Test with coverage
          run: dotnet test --no-build --verbosity normal --collect:"XPlat Code Coverage" --results-directory ./coverage
          working-directory: ./api
        
        - name: Generate coverage report
          uses: danielpalme/ReportGenerator-GitHub-Action@5.2.0
          with:
            reports: './api/coverage/**/coverage.cobertura.xml'
            targetdir: './api/coverage-report'
            reporttypes: 'HtmlInline;Cobertura'
        
        - name: Coverage threshold check
          run: |
            coverage=$(grep -oP 'line-rate="\K[0-9.]+' ./api/coverage-report/Cobertura.xml | head -1)
            threshold=0.80
            if (( $(echo "$coverage < $threshold" | bc -l) )); then
              echo "Coverage $coverage is below threshold $threshold"
              exit 1
            fi
  ```
- `.github/workflows/frontend-ci.yml` template:
  ```yaml
  name: Frontend CI

  on:
    push:
      branches: ['**']
    pull_request:
      branches: [main, develop]

  jobs:
    build-and-test:
      runs-on: ubuntu-latest

      steps:
        - uses: actions/checkout@v4
        
        - name: Setup Node.js
          uses: actions/setup-node@v4
          with:
            node-version: 18
            cache: 'npm'
            cache-dependency-path: './client/package-lock.json'
        
        - name: Install dependencies
          run: npm ci
          working-directory: ./client
        
        - name: Build
          run: npm run build
          working-directory: ./client
        
        - name: Test with coverage
          run: npm run test:ci -- --coverage --coverageReporters="text" --coverageReporters="cobertura"
          working-directory: ./client
        
        - name: Coverage threshold check
          run: |
            coverage=$(grep -oP 'line-rate="\K[0-9.]+' ./client/coverage/cobertura-coverage.xml)
            threshold=0.80
            if (( $(echo "$coverage < $threshold" | bc -l) )); then
              echo "Coverage $coverage is below threshold $threshold"
              exit 1
            fi
  ```
- Update `package.json` test:ci script: `"test:ci": "jest --ci --coverage --maxWorkers=2"`
