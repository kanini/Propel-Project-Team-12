# User Story - US_008

## Story ID

* ID: US_008

## Story Title

* Test Infrastructure Setup

## Description

* As a developer, I want xUnit (backend), Jest/Vitest (frontend), and Playwright (E2E) test frameworks configured with code coverage reporting targeting 80%, so that all layers have automated testing from the start.

## Acceptance Criteria

1. **Given** xUnit is the backend test framework, **When** I run `dotnet test`, **Then** the `PatientAccess.Tests` project executes with code coverage collection enabled and a coverage report is generated.

2. **Given** Vitest is configured for frontend, **When** I run `npm run test`, **Then** unit tests execute for React components and Redux slices with coverage output showing line, branch, and function metrics.

3. **Given** Playwright is the E2E framework, **When** I run Playwright tests, **Then** the test runner launches browser instances (Chromium at minimum) and a `playwright.config.ts` file configures base URL, timeouts, and retry settings.

4. **Given** the coverage target is 80%, **When** the CI pipeline runs tests, **Then** the pipeline fails if code coverage drops below 80% for business logic components.

## Edge Cases

* What happens when Playwright browsers are not installed? Setup script should include `npx playwright install` as prerequisite step.
* How does the system handle flaky E2E tests? Playwright config should include 1 retry for failed tests with test isolation ensured.
