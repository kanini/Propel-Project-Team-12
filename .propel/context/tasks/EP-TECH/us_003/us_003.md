# User Story - US_003

## Story ID

* ID: US_003

## Story Title

* Database Provisioning with PostgreSQL and pgvector

## Description

* As a developer, I want PostgreSQL 16 with the pgvector extension provisioned on Supabase with Entity Framework Core configured, so that the data layer is ready for schema migrations and vector storage.

## Acceptance Criteria

1. **Given** Supabase is the database host, **When** the database is provisioned, **Then** a PostgreSQL 16 instance is running with connection string stored in environment configuration (not source code).

2. **Given** pgvector extension is required, **When** I connect to the database, **Then** `CREATE EXTENSION IF NOT EXISTS vector` has been applied and 1536-dimensional vector columns can be created.

3. **Given** Entity Framework Core is the ORM, **When** I inspect `PatientAccess.Data`, **Then** a `PatientAccessDbContext` class exists inheriting from `DbContext` with proper connection string resolution from configuration.

4. **Given** the database is provisioned, **When** I run `dotnet ef migrations add InitialCreate`, **Then** a migration file is generated without errors and can be applied with `dotnet ef database update`.

## Edge Cases

* What happens when database connection fails at startup? Application should log connection error and fail fast with descriptive message.
* How does the system handle pgvector extension not available? Migration should fail with clear error indicating pgvector requirement.
