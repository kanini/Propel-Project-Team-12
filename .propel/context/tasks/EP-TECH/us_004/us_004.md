# User Story - US_004

## Story ID
* ID: US_004

## Story Title
* Database Configuration (PostgreSQL + EF Core)

## Description
* As a backend developer, I want Entity Framework Core configured with PostgreSQL connection and an initial empty database migration, so that the data layer foundation is ready for domain entity implementation.

## Acceptance Criteria
1. **Given** the .NET 8 API project exists, **When** I inspect the `.csproj` file, **Then** I find NuGet packages:
   - `Npgsql.EntityFrameworkCore.PostgreSQL` (version 8.x)
   - `Microsoft.EntityFrameworkCore.Design` (version 8.x)
   - `Microsoft.EntityFrameworkCore.Tools` (version 8.x)

2. **Given** EF Core is installed, **When** I review the `/Infrastructure` folder, **Then** I find:
   - `ApplicationDbContext.cs` class inheriting from `DbContext`
   - Connection string configuration reading from `appsettings.json` with key `"ConnectionStrings:DefaultConnection"`
   - `Program.cs` configures `services.AddDbContext<ApplicationDbContext>()` with PostgreSQL provider

3. **Given** the DbContext is configured, **When** I run `dotnet ef migrations add InitialCreate`, **Then** a migration file is generated in `/Infrastructure/Data/Migrations` folder with empty Up/Down methods.

4. **Given** the initial migration exists, **When** I run `dotnet ef database update`, **Then** the command executes successfully (even if no database is running yet), and EF Core tools validate the migration structure.

## Edge Cases
* What happens when the PostgreSQL connection string is invalid or missing? (Application logs clear error message and fails gracefully on startup with configuration exception)
* How does the project handle migration conflicts when multiple developers create migrations simultaneously? (Document migration naming conventions and version control merge strategy in README)

## Technical Notes
- Install EF Core CLI tools globally: `dotnet tool install --global dotnet-ef`
- Connection string format in `appsettings.json`:
  ```json
  {
    "ConnectionStrings": {
      "DefaultConnection": "Host=localhost;Database=PatientAccessDb;Username=postgres;Password=yourpassword"
    }
  }
  ```
- `ApplicationDbContext.cs` template:
  ```csharp
  public class ApplicationDbContext : DbContext
  {
      public ApplicationDbContext(DbContextOptions<ApplicationDbContext> options)
          : base(options)
      {
      }

      protected override void OnModelCreating(ModelBuilder modelBuilder)
      {
          base.OnModelCreating(modelBuilder);
          // Entity configurations will be added in EP-DATA
      }
  }
  ```
- Register DbContext in `Program.cs`:
  ```csharp
  builder.Services.AddDbContext<ApplicationDbContext>(options =>
      options.UseNpgsql(builder.Configuration.GetConnectionString("DefaultConnection")));
  ```
- Store sensitive connection strings in environment variables or user secrets for local development
