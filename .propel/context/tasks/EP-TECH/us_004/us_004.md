# User Story - US_004

## Story ID

* ID: US_004

## Story Title

* Authentication Infrastructure with JWT and BCrypt

## Description

* As a developer, I want JWT Bearer authentication with RS256 signing, BCrypt password hashing, and CORS policy configured, so that the security foundation is established for all API endpoints.

## Acceptance Criteria

1. **Given** JWT is configured, **When** I inspect the authentication middleware in `Program.cs`, **Then** JWT Bearer authentication is registered with RS256 signing algorithm and token validation parameters include issuer, audience, and lifetime validation.

2. **Given** BCrypt hashing is implemented, **When** a password is hashed using the auth service, **Then** the hash is generated with cost factor 12 and can be verified against the original password.

3. **Given** CORS policy is configured, **When** a request arrives from a non-allowed origin, **Then** the request is rejected with 403 status; requests from configured frontend domains are allowed.

4. **Given** token generation is implemented, **When** a valid user authenticates, **Then** a JWT token is returned containing user ID, email, and role claims with configurable expiration.

5. **Given** the auth middleware is active, **When** an unauthenticated request hits a protected endpoint, **Then** the API returns 401 Unauthorized with no sensitive information in the response body.

## Edge Cases

* What happens when the RS256 key pair is missing from configuration? Application should fail to start with clear error about missing signing keys.
* How does the system handle expired tokens? API returns 401 with `token_expired` error code distinguishable from invalid token.
