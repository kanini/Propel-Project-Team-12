# User Story - US_006

## Story ID
   * ID Format: US_006

## Story Title
   * JWT Authentication Infrastructure with RS256

## Description
  * As a developer, I want to implement JWT Bearer token authentication using RS256 asymmetric signing, so that API requests are securely authenticated with tokens that can be validated without exposing the signing key to frontend clients.

## Acceptance Criteria
  * **Given** the backend has JWT middleware configured, **When** the application starts, **Then** RSA key pair (public/private) is loaded from secure configuration for RS256 signing algorithm.

  * **Given** a user successfully authenticates, **When** the AuthService generates a JWT token, **Then** the token includes user ID, email, role claims, issued-at time, and 15-minute expiration signed with RS256 private key.

  * **Given** an API request includes a JWT token in Authorization header, **When** the middleware validates the token, **Then** the token signature is verified using the RS256 public key and claims are extracted for authorization.

  * **Given** the JWT token is expired, **When** the middleware validates it, **Then** the request is rejected with 401 Unauthorized and a clear error message indicating token expiration.

  * **Given** the JWT token is tampered with, **When** signature validation is performed, **Then** the request is rejected with 401 Unauthorized and the security event is logged to audit trail.

  * **Given** role-based claims are in the JWT token, **When** an endpoint requires specific role (e.g., [Authorize(Roles="Staff")]), **Then** only users with matching role claim can access the endpoint per NFR-006.

## Edge Cases
   * What happens when the RSA private key file is missing or corrupted? (Application fails to start; log fatal error with configuration path)
   * How does token validation handle clock skew between servers? (Configure 5-minute clock skew tolerance in JWT validation parameters)
   * What if the Authorization header format is invalid? (Return 401 with clear message "Invalid Authorization header format; use Bearer <token>")
