# User Story - US_006

## Story ID
* ID: US_006

## Story Title
* Backend Testing Framework Setup

## Description
* As a backend developer, I want xUnit, FluentAssertions, and Testcontainers configured in a test project, so that I can write unit and integration tests with expressive assertions and containerized database testing.

## Acceptance Criteria
1. **Given** the `/api` solution exists, **When** I inspect the solution structure, **Then** I find a test project named `API.Tests` or similar with:
   - `xUnit` (version 2.x) as test runner
   - `FluentAssertions` (version 6.x) for readable assertions
   - `Testcontainers` (version 3.x) for containerized PostgreSQL testing
   - `Microsoft.AspNetCore.Mvc.Testing` for integration testing

2. **Given** the test project is configured, **When** I run `dotnet test` from the `/api` folder, **Then** the test runner executes successfully with 0 tests (initial state) and displays summary output.

3. **Given** the testing framework is set up, **When** I create a sample unit test using FluentAssertions:
   ```csharp
   [Fact]
   public void SampleTest_Should_Pass()
   {
       // Arrange
       var expected = 5;
       // Act
       var actual = 2 + 3;
       // Assert
       actual.Should().Be(expected);
   }
   ```
   **Then** the test passes when executed with `dotnet test`.

4. **Given** Testcontainers is configured, **When** I create a sample integration test that spins up a PostgreSQL container using `Testcontainers.PostgreSql`, **Then** the test successfully connects to the containerized database and executes a simple query.

## Edge Cases
* What happens when Docker is not running during integration test execution? (Tests throw descriptive error: "Docker daemon not reachable" with guidance to start Docker Desktop)
* How does the test project handle slow Testcontainers startup on low-resource machines? (Configure reasonable timeout values and document minimum hardware requirements)

## Technical Notes
- Create test project: `dotnet new xunit -n API.Tests`
- Install NuGet packages in test project:
  ```bash
  dotnet add package xUnit
  dotnet add package FluentAssertions
  dotnet add package Testcontainers.PostgreSql
  dotnet add package Microsoft.AspNetCore.Mvc.Testing
  ```
- Sample Testcontainers integration test:
  ```csharp
  public class DatabaseIntegrationTests : IAsyncLifetime
  {
      private PostgreSqlContainer _postgresContainer;

      public async Task InitializeAsync()
      {
          _postgresContainer = new PostgreSqlBuilder()
              .WithImage("pgvector/pgvector:pg16")
              .Build();
          await _postgresContainer.StartAsync();
      }

      [Fact]
      public async Task Database_Should_Connect()
      {
          var connectionString = _postgresContainer.GetConnectionString();
          // Test database connection
          connectionString.Should().NotBeNullOrEmpty();
      }

      public async Task DisposeAsync()
      {
          await _postgresContainer.DisposeAsync();
      }
  }
  ```
- Configure test project to reference main API project
- Document test execution commands in README
