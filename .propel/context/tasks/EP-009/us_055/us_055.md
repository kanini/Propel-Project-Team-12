# User Story - US_055

**Status:** ✅ **COMPLETE** (March 30, 2026)

## Story ID

* ID: US_055

## Story Title

* Verification Queue - Patient Selection

## Description

* As a Staff member, I want to see a prioritized list of all patients with pending verifications so that I can select which patient to verify based on urgency and data conflicts.

## Acceptance Criteria

1. **Given** I navigate to the Verification section from the staff dashboard, **When** the verification queue loads, **Then** I see a table displaying all patients with pending clinical data or medical code verifications, sorted by priority (High, Medium, Low).

2. **Given** I am viewing the verification queue, **When** I look at each patient row, **Then** I see: Patient Name, MRN, count of pending clinical data items, count of pending medical codes, count of data conflicts, last upload date, and priority indicator.

3. **Given** I want to filter the queue, **When** I use the filter controls, **Then** I can filter by: Priority level (All/High/Medium/Low), Document Type (All/Lab Results/Imaging/etc.), and Date range (All/Today/This Week/This Month).

4. **Given** I want to find a specific patient, **When** I type in the search box, **Then** the system performs real-time search by patient name or MRN and updates the table results.

5. **Given** I want to verify a specific patient's data, **When** I click the "Review" button or click anywhere on the patient row, **Then** the system navigates to SCR-023 (Clinical Verification screen) for that specific patient.

6. **Given** the priority requirements (FR-041), **When** the system calculates priority, **Then** priority is automatically determined based on: number of conflicts (highest weight), pending data volume, and time since upload.

7. **Given** I am viewing the queue, **When** the page loads, **Then** I see a summary stat showing total count of patients with pending verifications (e.g., "8 patients pending verification").

8. **Given** accessibility requirements (WCAG 2.1 AA), **When** I navigate the queue, **Then** all interactive elements are keyboard accessible, ARIA labels are present, and screen readers can announce table data correctly.

## Edge Cases

* What happens when a patient has no conflicts but has pending verifications? They appear in the list with a conflict count of 0 and priority calculated based on pending volume and time.
* How does the system handle when two staff members open the same patient for verification? The system allows concurrent review but uses optimistic locking on save - the second save receives a conflict notification if the first staff member already modified the same data point.
* What happens when all verifications for a patient are completed while the queue is open? The patient row is removed from the queue on next refresh or real-time update (if implemented).
* How does priority change when conflicts are resolved? Priority recalculates automatically; if all conflicts are resolved, the patient moves from High to Medium/Low priority based on remaining pending items.

## Business Rules

* **BR-041**: Priority calculation formula: Priority = (Conflicts × 10) + (Pending Items × 2) + (Days Since Upload × 1). High >= 30, Medium >= 15, Low < 15.
* **BR-042**: Search performs partial match on Patient First Name, Last Name, and MRN.
* **BR-043**: Queue refresh interval: Real-time via WebSocket OR 30-second polling fallback.

## Technical Notes

### Frontend Implementation

* **Component**: `VerificationQueue.tsx`
* **State Management**: TanStack Query for server state, local filters via useState
* **API Endpoint**: GET `/api/verification/queue` with query params: `priority`, `documentType`, `dateRange`, `searchTerm`

### Backend Implementation

* **Service**: `VerificationQueueService.cs`
* **Database Query**: Joins `PatientDocuments`, `ExtractedClinicalData`, `MedicalCodes` tables; filters by `VerificationStatus = Pending`; calculates priority score; supports pagination
* **Response DTO**: `VerificationQueueItemDto` with fields: PatientId, PatientName, MRN, ClinicalDataCount, MedicalCodeCount, ConflictCount, LastUploadDate, Priority
