# User Story - US_044

## Story ID

* ID: US_044

## Story Title

* Document Processing Status Tracking

## Description

* As a patient, I want to view the processing status of my uploaded documents (uploaded, processing, completed, failed), so that I know when my clinical data is ready.

## Acceptance Criteria

1. **Given** I have uploaded documents, **When** I navigate to the Document Status page, **Then** all my documents are listed with name, upload date, file size, and current processing status.

2. **Given** a document is being processed, **When** the status changes, **Then** the status updates in real-time via Pusher Channels without requiring page refresh, and ARIA live regions announce the change for accessibility (UXR-207).

3. **Given** a document has completed processing, **When** I view its status, **Then** a "View Extracted Data" link appears allowing me to see the data extracted from the document.

4. **Given** no documents have been uploaded, **When** the page loads, **Then** an empty state with illustration and "Upload your first document" CTA is displayed.

## Edge Cases

* What happens when a document is stuck in "Processing" for over 5 minutes? System should show "Processing is taking longer than expected" message with contact support option.
* How does the system handle documents that fail extraction? Status shows "Failed" with a "Retry" button that re-queues the processing job.
