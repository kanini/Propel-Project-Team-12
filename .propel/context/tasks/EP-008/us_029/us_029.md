# User Story - US_029

## Story ID
* ID: US_029

## Story Title
* RBAC Middleware and API Authorization

## Description
* As a backend API, I want to enforce role-based access control on all API endpoints, so that users can only access features appropriate to their assigned role (Patient/Staff/Admin) and unauthorized requests are blocked.

## Acceptance Criteria
1. **Given** an API endpoint is decorated with `[Authorize(Roles = "Staff")]` attribute, **When** a Patient user attempts to access the endpoint, **Then** the RBAC middleware extracts the role claim from the JWT token, validates against required roles, and rejects the request with 403 Forbidden status and error message "Insufficient permissions."

2. **Given** a Staff user accesses a Staff-only endpoint, **When** the middleware validates the JWT token, **Then** the system extracts the `role` claim, verifies it matches "Staff," allows the request to proceed to the controller, and logs the authorized access in the audit log.

3. **Given** an endpoint requires multiple role permissions, **When** the `[Authorize(Roles = "Staff,Admin")]` attribute is applied, **Then** the middleware allows access if the user's role matches any of the specified roles (OR logic), and denies access otherwise.

4. **Given** an API controller method checks fine-grained permissions, **When** the controller accesses `User.Claims`, **Then** the system provides the `grantedPermissions` array claim, enabling permission-level authorization beyond role checks (e.g., "CanViewAuditLogs" permission within Admin role).

## Edge Cases
* What happens when a user's role is changed while they have an active JWT token? (Token remains valid with old role until expiration; new login required for updated role; deactivation flow invalidates session via blacklist if implemented)
* How does the system handle endpoints with no `[Authorize]` attribute? (Endpoints without attribute are publicly accessible; middleware logs warning during startup if sensitive endpoint lacks authorization; security scanning detects missing attributes)
* What happens when JWT token is missing from the request? (Middleware returns 401 Unauthorized with message "Authentication required"; request does not reach controller)
* How does the system enforce database row-level security alongside RBAC? (Repository layer filters queries by `userID` or `assignedPatients` based on role; Patient can only query own records; Staff queries filtered by assigned patients; Admin unrestricted)
