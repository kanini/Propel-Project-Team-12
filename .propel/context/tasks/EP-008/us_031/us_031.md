# User Story - US_031

## Story ID
* ID: US_031

## Story Title
* Admin User Management List UI

## Description
* As an Admin, I want to view a searchable, filterable list of all user accounts, so that I can quickly find users and perform management actions like editing, deactivating, or resetting passwords.

## Acceptance Criteria
1. **Given** I am logged in as an Admin, **When** I navigate to the "User Management" page, **Then** the system displays a table showing all users with columns (Email, Name, Role, Status, Created Date) sorted by created date descending, and loads data via `GET /api/admin/users` returning active and inactive users.

2. **Given** the user list is displayed, **When** I type in the search box, **Then** the system filters users client-side by email or name (case-insensitive partial match), updates the table in real-time as I type, and displays "No users found" if no matches exist.

3. **Given** the user list is displayed, **When** I select a role filter (Patient, Staff, Admin, All), **Then** the system filters the table to show only users with the selected role, and displays the count of filtered users ("Showing 45 of 120 users").

4. **Given** the user table displays many users, **When** there are more than 50 users, **Then** the system implements client-side pagination with 50 users per page, provides "Previous" and "Next" buttons, and displays current page number ("Page 2 of 3").

## Edge Cases
* What happens when I search for a user while a filter is active? (Search applies to already-filtered results; clearing search restores filtered view, not full list)
* How does the system handle very long user names or emails in the table? (Text truncates with ellipsis after 30 characters; hover shows full value in tooltip; full value visible in detail view)
* What happens if the API returns an error when loading users? (Display error message "Failed to load users. Please try again" with retry button; log error details; fallback to cached data if available)
* How does the system indicate users who have never logged in? (Display "Never" in Last Login column; badge shows "Pending Activation" status)
