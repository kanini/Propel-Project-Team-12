# User Story - US_050

## Story ID

* ID: US_050

## Story Title

* RAG Knowledge Base Setup

## Description

* As a developer, I want a RAG knowledge base with chunked medical coding documents, separate vector indices for ICD-10, CPT, and clinical terminology, and hybrid retrieval, so that the medical coding pipeline has an accurate grounding layer.

## Acceptance Criteria

1. **Given** medical coding documents are available, **When** the indexing pipeline runs, **Then** documents are chunked into 512-token segments with 64-token overlap (12.5%) per AIR-R01.

2. **Given** chunks are created, **When** embeddings are generated, **Then** Azure OpenAI text-embedding-3-small produces 1536-dimensional vectors stored in separate pgvector indices for ICD-10, CPT, and clinical terminology (AIR-R04).

3. **Given** a code mapping query is submitted, **When** retrieval executes, **Then** top-5 chunks with cosine similarity above 0.75 are returned (AIR-R02) using hybrid retrieval combining semantic similarity and keyword matching (AIR-R03).

4. **Given** the knowledge base is populated, **When** I query for a clinical term like "Type 2 Diabetes Mellitus", **Then** relevant ICD-10 codes (e.g., E11.x) and related CPT codes are retrieved with similarity scores.

## Edge Cases

* What happens when a query term has no matches above 0.75 threshold? System returns "No confident matches found" and falls back to keyword-only search with reduced confidence scoring.
* How does the system handle knowledge base updates (new code releases)? Re-indexing pipeline can be triggered independently without affecting live queries.
