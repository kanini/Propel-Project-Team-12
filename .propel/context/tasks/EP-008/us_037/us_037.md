# User Story - US_037

## Story ID
* ID: US_037

## Story Title
* Session Timeout Implementation

## Description
* As a user of any role, I want my session to automatically timeout after 15 minutes of inactivity with a 2-minute warning, so that my account remains secure if I leave my workstation unattended while preserving unsaved work temporarily.

## Acceptance Criteria
1. **Given** I am logged in and using the application, **When** the React inactivity tracker detects 13 minutes of no user activity (no clicks, keyboard input, or API calls), **Then** the system displays a modal dialog with message "You'll be logged out in 2 minutes due to inactivity. Continue session?" and a "Stay Logged In" button overlaying the current screen.

2. **Given** the timeout warning modal is displayed, **When** I click the "Stay Logged In" button within 2 minutes, **Then** the system calls `POST /api/auth/refresh` to generate a new JWT token with updated expiration, updates the httpOnly cookie with the new token, resets the inactivity timer to 0, closes the warning modal, and I can continue working normally.

3. **Given** the timeout warning modal is displayed, **When** I do not respond within 2 minutes (total 15 minutes inactivity), **Then** the system automatically logs me out by calling `POST /api/auth/logout` to invalidate the session, clears the JWT cookie, saves unsaved form data to secure session storage (localStorage encrypted) with 1-hour expiration, redirects to the login page with message "Logged out due to inactivity," and logs the timeout event in audit log.

4. **Given** I was automatically logged out with unsaved form data, **When** I log back in within 1 hour, **Then** the system checks session storage for preserved data, displays a notification "You have unsaved work. Restore it?" with "Restore" and "Discard" buttons, and restores the form data to the appropriate screen if I click "Restore."

## Edge Cases
* What happens if I have multiple tabs open and one tab times out? (All tabs share the same JWT cookie; timeout in one tab triggers logout across all tabs; all tabs redirect to login simultaneously; session storage preserved once)
* How does the system track activity across API calls? (Each API request resets the inactivity timer; background API calls (e.g., polling) do NOT reset timer to prevent artificially extending sessions; only user-initiated requests reset timer)
* What happens if the network is unavailable when the refresh token call is made? (Refresh call times out after 5 seconds; modal displays network error; user can retry "Stay Logged In" or will be logged out when 15 minutes expires; audit log records attempted refresh failure)
* How does the system handle saving unsaved data that contains PHI? (Session storage data encrypted using Web Crypto API with key derived from JWT token; data purged after 1 hour absolute or on successful logout; audit log records data preservation event)
