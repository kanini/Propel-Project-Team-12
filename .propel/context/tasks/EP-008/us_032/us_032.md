# User Story - US_032

## Story ID
* ID: US_032

## Story Title
* Admin Create User Workflow

## Description
* As an Admin, I want to create new user accounts with email, name, role, and permissions, so that I can onboard new patients and staff members efficiently and they receive activation emails to set their password.

## Acceptance Criteria
1. **Given** I am on the User Management page, **When** I click the "Create User" button, **Then** the system displays a modal form with fields (Email, Name, Role dropdown: Patient/Staff/Admin, Permissions checkboxes based on role), validation indicators, and "Create" and "Cancel" buttons.

2. **Given** the create user form is displayed, **When** I enter user details and select role "Staff," **Then** the system dynamically shows Staff-specific permission checkboxes (ManageWalkIns, ReviewClinicalData, VerifyExtractions), and hides Patient/Admin-specific permissions.

3. **Given** I submit the create user form with valid data, **When** the system calls `POST /api/admin/users`, **Then** the backend validates email uniqueness, creates the user account with status "Pending Activation," generates a temporary password, sends an activation email with the temporary password and activation link, and returns 201 Created with the new user ID.

4. **Given** the user is successfully created, **When** the API returns success, **Then** the modal closes, the user list refreshes to include the new user, and a success toast displays "User created successfully. Activation email sent to {email}."

## Edge Cases
* What happens when I try to create a user with an email that already exists? (API returns 409 Conflict with error "User with this email already exists"; form displays error message under email field; user list does not refresh)
* How does the system handle activation email delivery failures? (Email service returns bounce/failure; system marks user status as "Pending Activation (Email Failed)"; admin sees warning icon in user list; admin can manually resend activation email)
* What happens if I close the modal without saving? (System prompts "Discard unsaved changes?" if form has data; clicking "Yes" closes modal and discards data; clicking "No" returns to form)
* How does the permissions checkbox selection work when role changes? (Changing role clears previously selected permissions and loads role-appropriate permissions; system warns "Changing role will reset permissions. Continue?")
