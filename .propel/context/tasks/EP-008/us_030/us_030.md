# User Story - US_030

## Story ID
* ID: US_030

## Story Title
* Role-Based Navigation UI

## Description
* As a user of any role (Patient/Staff/Admin), I want to see only the navigation menu items appropriate to my role, so that I am not confused by features I cannot access and can quickly find the functions I need.

## Acceptance Criteria
1. **Given** a Patient user logs in successfully, **When** the application loads the navigation menu, **Then** the UI displays only Patient-appropriate items (Dashboard, Book Appointment, Intake Forms, My Documents) and hides Staff and Admin items (Walk-Ins, Arrivals, Clinical Review, User Management, Audit Logs).

2. **Given** a Staff user logs in successfully, **When** the navigation menu renders, **Then** the UI displays Staff items (Dashboard, Arrivals, Walk-Ins, Clinical Review, Coding Review) and hides Patient-specific items (Book Appointment, My Intake) and Admin items (User Management, Audit Logs, System Configuration).

3. **Given** an Admin user logs in successfully, **When** the navigation menu renders, **Then** the UI displays all Admin items (Dashboard, User Management, System Configuration, Audit Logs) and hides Patient and Staff items unless Admin also has Staff permissions.

4. **Given** the user's role is stored in React Context from JWT claims, **When** a protected route is accessed via URL manipulation, **Then** the application checks the role from context, denies access if role does not match route requirements, and redirects to appropriate dashboard with error toast "Access denied."

## Edge Cases
* What happens when a user's session expires while navigating? (React checks JWT expiration before each route change, redirects to login if expired, preserves attempted route for post-login redirect)
* How does the UI handle Admin users who also have Staff role? (Navigation displays union of permissions; context checks highest privilege role; Admin sees both Admin and Staff nav items)
* What happens if JWT token is tampered to change role claim? (Backend API rejects tampered token with 401; frontend detects API rejection, clears session, redirects to login)
* How does the system handle navigation when user opens multiple tabs with different roles? (Each tab maintains separate React Context; JWT cookie shared across tabs; role mismatch detection logs out all tabs and forces re-authentication)
