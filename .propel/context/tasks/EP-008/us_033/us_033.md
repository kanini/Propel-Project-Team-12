# User Story - US_033

## Story ID
* ID: US_033

## Story Title
* Admin Edit User Details

## Description
* As an Admin, I want to edit existing user account details (name, role, permissions), so that I can update user information when roles change or correct errors without creating new accounts.

## Acceptance Criteria
1. **Given** I am viewing the user management list, **When** I click the "Edit" button for a user, **Then** the system opens the edit user modal pre-populated with current user data (email read-only, name, role, permissions), and displays "Save Changes" and "Cancel" buttons.

2. **Given** the edit user form is displayed, **When** I modify the user's role from "Patient" to "Staff," **Then** the system prompts "Changing role will reset permissions. Continue?", clears existing permissions upon confirmation, displays Staff-specific permission checkboxes, and allows me to assign new permissions.

3. **Given** I modify user details and click "Save Changes," **When** the system calls `PUT /api/admin/users/{id}`, **Then** the backend validates the changes, updates the user record, logs the modification in the audit log with admin ID and timestamp, and returns 200 OK with updated user data.

4. **Given** the user update is successful, **When** the API responds, **Then** the modal closes, the user list refreshes to reflect updated information, and a success toast displays "User updated successfully."

## Edge Cases
* What happens when I try to change a user's email address? (Email field is read-only in edit mode; error message displays "Email cannot be changed. Create a new user or contact support to merge accounts")
* How does the system handle editing a user who has active sessions? (Edit succeeds; role/permission changes do not affect active JWT tokens; user must log out and log back in for changes to take effect; system can optionally invalidate sessions via concurrent session blocking)
* What happens if another admin deactivates the user while I'm editing? (On save, API returns 409 Conflict "User status changed by another admin"; form displays error; admin can refresh form data or cancel)
* How does the system handle permission conflicts during edit? (Validation ensures Admin role has all permissions enabled and cannot be disabled; attempting to remove critical Admin permissions shows error "Cannot remove required Admin permissions")
