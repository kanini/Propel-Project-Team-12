# User Story - US_036

## Story ID
* ID: US_036

## Story Title
* User Account Activation Flow

## Description
* As a newly created user, I want to activate my account by setting my password via an activation link, so that I can establish secure credentials and access the system without the admin knowing my password.

## Acceptance Criteria
1. **Given** an admin creates my user account, **When** the activation email is sent, **Then** I receive an email containing my temporary password, an activation link with a unique activation token, and instructions "Click the link to activate your account and set a new password."

2. **Given** I click the activation link in the email, **When** the system validates the activation token against the PasswordResetTokens table, **Then** the system checks token validity (not expired, matches userID, not already used), displays the account activation form if valid, or shows "Activation link expired or invalid" error if invalid.

3. **Given** the activation form is displayed, **When** I enter and confirm my new password meeting password policy requirements (min 8 chars, 1 uppercase, 1 lowercase, 1 digit, 1 special char), **Then** the system validates the password, hashes it with bcrypt (work factor 12), updates the user record, marks the activation token as consumed, updates account status to "Active," and returns 200 OK.

4. **Given** my account is successfully activated, **When** the activation completes, **Then** the system displays "Account activated successfully! Redirecting to login..." message, logs the activation event in the audit log with userID and timestamp, and redirects me to the login page after 3 seconds.

## Edge Cases
* What happens if I try to activate an account that was already activated? (Token marked as consumed; activation link returns "Account already activated. Please log in" with redirect to login page)
* How does the system handle activation links that expire before the user clicks them? (Token expiration set to 7 days for activation vs 1 hour for password reset; expired link shows error with "Request new activation email" button triggering admin notification)
* What happens if I enter a weak password that doesn't meet policy requirements? (Client-side validation displays real-time error messages ("Password must contain uppercase letter," etc.); form submission disabled until all requirements met; backend validates again and rejects with 400 Bad Request if bypassed)
* How does the system handle users who lose the activation email? (User contacts admin; admin can resend activation email via "Resend Activation" button in user management list; new token generated, old token invalidated)
