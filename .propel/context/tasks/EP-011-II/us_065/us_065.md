# User Story - US_065

## Story ID

* ID: US_065

## Story Title

* Interaction Feedback & Loading States

## Description

* As a Patient, I want immediate visual feedback on every interaction (clicks, submissions, loading) with clear loading states and progress indicators, so that I understand the system is responding to my actions.

## Acceptance Criteria

1. **Given** action feedback (UXR-501), **When** I click a button or submit a form, **Then** visual feedback (button state change, spinner, or checkmark) appears within 200ms of the interaction.

2. **Given** loading states (UXR-502), **When** data is being fetched, **Then** skeleton loaders display in the content area matching the expected layout shape, and a text label ("Loading appointments...") is present for screen readers.

3. **Given** progress indicators (UXR-504), **When** a long-running operation occurs (document upload, AI processing), **Then** a progress bar or step indicator shows percentage or stage completion with estimated time remaining when available.

4. **Given** optimistic UI updates, **When** I perform a quick action (toggle, mark complete), **Then** the UI updates immediately and rolls back with an error toast if the server request fails.

## Edge Cases

* What happens when a loading state exceeds 10 seconds? A "Still working..." message replaces the initial loading text, and after 30 seconds a "Something may have gone wrong" message with a retry button appears.
* How are multiple concurrent loading states handled? Each content section manages its own skeleton loader independently; a global loading bar appears only for full-page navigations.
