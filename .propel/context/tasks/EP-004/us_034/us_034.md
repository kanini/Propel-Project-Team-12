# User Story - US_034

## Story ID

- ID: US_034

## Story Title

- Manual Form Intake

## Description

- As a patient, I want to complete my pre-visit intake using a traditional structured form, so that I can provide my health information through a familiar form-based interface.

## Acceptance Criteria

1. **Given** I have selected a specific appointment from the appointment list (US_037), **When** I select "Manual Form" on the intake page, **Then** a structured intake form displays with sections for medical history, current medications, allergies, visit concerns, and insurance information for that specific appointment.

2. **Given** I am filling out the form, **When** I complete required fields and submit, **Then** all intake data is saved as an IntakeRecord linked to my appointment with mode set to "Manual".

3. **Given** field validation is active, **When** I leave required fields blank or enter invalid data, **Then** inline validation errors display below the corresponding fields with specific correction instructions.

4. **Given** the form has clinical terminology, **When** I hover over or focus on fields with medical terms, **Then** contextual help tooltips explain the terminology in plain language (UXR-103).

## Edge Cases

- What happens when the form submission fails mid-save? Entered data is preserved in browser local storage and restored on retry.
- How does the system handle extremely long text responses in free-text fields? Character limits are enforced (e.g., 2000 characters for visit concerns) with a counter visible.
