# User Story - US_036

## Story ID

* ID: US_036

## Story Title

* Insurance Pre-Check Validation

## Description

* As a patient, I want my insurance information validated against the system's internal reference records during intake, so that I know my coverage status before my appointment.

## Acceptance Criteria

1. **Given** I am completing the insurance section of intake, **When** I enter my insurance provider name and ID, **Then** the system validates the ID against the InsuranceRecord reference data using pattern matching (regex).

2. **Given** the insurance is found and valid, **When** validation succeeds, **Then** a green checkmark displays with coverage type information (e.g., "Insurance verified — PPO coverage").

3. **Given** the insurance ID does not match patterns, **When** validation fails, **Then** a warning message displays "Insurance could not be verified. Please check your insurance ID." with the option to proceed anyway.

4. **Given** the insurance provider is not in the reference data, **When** the provider name has no match, **Then** the system indicates "Provider not found in our records" and allows manual entry to continue.

## Edge Cases

* What happens when the insurance reference data is empty? System skips validation and logs warning; intake proceeds with "Validation unavailable" status.
* How does the system handle insurance IDs with leading zeros or special formats? Pattern matching uses regex defined in InsuranceRecord to accommodate varied ID formats.
