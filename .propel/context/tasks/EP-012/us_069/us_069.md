# User Story - US_069

## Story ID

* ID: US_069

## Story Title

* Patient Portal Screen Completion (Confirmation, Waitlist, Reschedule, Profile)

## Description

* As a patient, I want complete appointment management and profile screens (Confirmation, Waitlist, Reschedule, Profile) implemented, so that I can view booking confirmations, join waitlists, reschedule appointments, and manage my account settings through a complete user experience.

## Acceptance Criteria

1. **Given** I successfully book an appointment, **When** booking completes, **Then** the system navigates to Appointment Confirmation page (SCR-008) displaying appointment details, provider info, date/time, and action buttons for "Add to Calendar" (Google/Outlook), "View My Appointments", and "Back to Dashboard".

2. **Given** I attempt to book an unavailable time slot, **When** selecting a preferred unavailable slot, **Then** the system displays Waitlist Enrollment page (SCR-009) allowing me to select the preferred slot, set notification preferences, and join the waitlist with automatic swap capability when the slot becomes available.

3. **Given** I have an upcoming appointment, **When** I click "Reschedule" from My Appointments page, **Then** the system displays Reschedule Appointment page (SCR-011) with date/time picker, conflict detection, original appointment details, and confirmation workflow that updates the appointment and sends updated notifications.

4. **Given** I navigate to my profile, **When** I access the Profile page from mobile bottom navigation or sidebar, **Then** the system displays Patient Profile page with sections for: account settings (email, phone, password), personal information (name, DOB, gender, address), notification preferences (SMS/Email toggles), and accessibility settings (language, font size).

## Edge Cases

* What happens when calendar sync fails? System displays error message "Calendar sync unavailable. Please add appointment manually." with retry button and fallback to download .ics file.
* What happens when a preferred slot becomes available while the user is offline? System sends notification via configured channels (SMS/Email) and automatically swaps appointment when user is next online, with confirmation email sent.
* What happens if reschedule target conflicts with another appointment? System displays inline error "This time conflicts with your appointment on [date] at [time]. Please select a different time." and prevents submission.
* What happens when profile update fails due to duplicate email? System displays field-level error "This email is already registered. Please use a different email." below email field.
