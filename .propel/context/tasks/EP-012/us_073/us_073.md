# User Story - US_073

## Story ID

* ID: US_073

## Story Title

* Admin Portal Completion (Dashboard & System Settings)

## Description

* As a system administrator, I want a comprehensive Admin Dashboard (SCR-005) and System Settings page (SCR-026) to monitor platform health, manage system-wide configurations, and control feature toggles, so that I can oversee platform operations, configure integrations, and manage system behavior without code changes.

## Acceptance Criteria

1. **Given** I am an admin user, **When** I log in and access Admin Dashboard (SCR-005), **Then** the system displays dashboard with six metric panels: Total Users (Patient/Staff/Admin counts), Active Appointments (Today, This Week, This Month), Document Processing Stats (Uploaded, Processing, Completed, Failed), System Health (API response time, database status, cache status, circuit breaker status), Recent Activity Timeline (last 10 user registrations, appointments, document uploads), and Quick Actions panel (Create User, View Audit Log, Configure Settings).

2. **Given** I am viewing Admin Dashboard, **When** I click on any metric panel, **Then** the system navigates to the detailed view for that metric (e.g., Total Users → User Management page, Active Appointments → Appointments report, System Health → Health Check endpoint details).

3. **Given** I navigate to System Settings page (SCR-026), **When** the page loads, **Then** the system displays configuration sections: General Settings (platform name, support email, timezone), Feature Toggles (AI Intake enabled/disabled, Waitlist enabled/disabled, Calendar Sync enabled/disabled), Integration Settings (Pusher credentials, SendGrid API key, Twilio API key, Azure AI Document Intelligence endpoint), Notification Settings (default SMS/Email templates, reminder timing configuration), and Security Policies (session timeout duration, password complexity rules, JWT expiration).

4. **Given** I modify any system setting, **When** I click "Save Changes" button, **Then** the system validates configuration (e.g., credentials for integrations), persists changes to appsettings.json or database configuration table, logs the change in audit log with admin ID, timestamp, setting name, old value, new value (NFR-007), applies changes without requiring application restart (hot reload for eligible settings), and displays success toast "Settings saved successfully. Changes applied."

## Edge Cases

* What happens when admin enters invalid API credentials for integrations? System performs immediate validation (test API call), displays field-level error "Invalid credentials. Please check and try again." below the field, and prevents save until valid credentials provided.
* What happens when another admin modifies settings simultaneously? Use optimistic locking; if conflict detected, display warning "Settings were modified by [Admin Name] at [Time]. Please refresh and try again." with refresh button.
* What happens when settings save fails due to database error? Display error banner "Unable to save settings. Please try again." with retry button, preserve user's changes in UI state, and log error to Application Insights.
* What happens when admin tries to disable AI Intake while extraction jobs are running? Display confirmation modal "X document processing jobs are currently running. Disabling AI Intake will queue new uploads until enabled again. Continue?" with Confirm/Cancel buttons.
* What happens when non-admin user tries to access Admin Dashboard or Settings? Protected route redirects to role-appropriate dashboard (Patient → Patient Dashboard, Staff → Staff Dashboard) and logs unauthorized access attempt with 403 status (NFR-006, NFR-007).
