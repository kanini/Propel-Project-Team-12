# User Story - US_071

## Story ID

* ID: US_071

## Story Title

* Staff Patient View, Verification Queue & Clinical Verification Workflow

## Description

* As a staff member, I want a Verification Queue page (SCR-023A), comprehensive Staff Patient View (SCR-017), and Clinical Verification page (SCR-023) to review AI-extracted patient data, verify accuracy, and manage data validation workflows, so that I can efficiently triage patients requiring verification, ensure patient health information is accurate before clinical use, and complete the Trust-First verification workflow.

## Acceptance Criteria

1. **Given** I am a staff member, **When** I navigate to Verification from the sidebar, **Then** the system displays Verification Queue page (SCR-023A) showing a filterable table of all patients with pending verification items, including: patient name, MRN, priority level (High/Medium/Low based on conflict severity or document age), document type, pending items count, upload date, and "Review" action button per row.

2. **Given** I am viewing Verification Queue, **When** I use search or filter controls, **Then** the system filters the patient list by: search query (patient name or MRN), priority level dropdown, document type dropdown, or date range, and updates the patient count badge in real-time showing "X patients pending verification".

3. **Given** I am viewing Verification Queue, **When** I click "Review" button on a patient row, **Then** the system navigates to Clinical Verification page (SCR-023) for that specific patient, displaying all AI-suggested data grouped by category with confidence scores, source references, and bulk actions (Accept All, Review Individually).

4. **Given** I am a staff member viewing a specific patient, **When** I navigate to Staff Patient View (SCR-017), **Then** the system displays consolidated patient profile with: demographics, conflict alerts banner (if any), extracted clinical data sections (conditions, medications, allergies, vitals, encounters), verification badges (amber for AI-suggested, green for staff-verified), document history timeline, source document references for each data point, and "Verify Data" button that navigates to SCR-023.

5. **Given** I am on Clinical Verification page for a specific patient, **When** I review an individual data item, **Then** I can select one of three actions: Accept (marks as verified, badge turns green), Modify (opens inline edit form with original value pre-filled, saves modified value as verified), or Reject (removes from patient profile, logs rejection reason), and the system updates verification status with staff ID and timestamp (NFR-007).

6. **Given** I have completed verification actions, **When** I click "Save Verifications" button, **Then** the system persists all verification decisions, updates patient profile, removes patient from Verification Queue if all items verified, sends Pusher real-time update to patient dashboard (if patient is online), logs audit trail entry (NFR-007), displays success toast "X items verified successfully", and navigates back to Verification Queue (SCR-023A).

## Edge Cases

* What happens when multiple staff members verify the same patient simultaneously? System uses optimistic locking with last-write-wins strategy; if conflict detected, display warning "Another staff member updated this patient's data. Please refresh and try again." with refresh button.
* What happens when verification API call fails? Display error banner "Unable to save verification. Please try again." with retry button, and preserve user's selections in UI state to prevent data loss.
* What happens when patient uploads new document while staff is verifying? Pusher notification displays banner "New document uploaded for this patient. Refresh to see latest data." with refresh button, and patient re-appears in Verification Queue with updated pending count.
* What happens when staff tries to verify data but lacks Staff/Admin role? Protected route redirects to appropriate dashboard with 403 error logged (NFR-006).
* What happens when Verification Queue has zero pending patients? Display empty state with illustration "No patients requiring verification. Great work!" and count shows "0 patients pending verification".
* What happens when staff searches Verification Queue but no results found? Display empty state "No patients match your search. Try different filters." with "Clear Filters" button.
