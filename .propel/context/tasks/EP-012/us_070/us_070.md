# User Story - US_070

## Story ID

* ID: US_070

## Story Title

* Patient Health Dashboard with 360-Degree Patient View

## Description

* As a patient, I want a comprehensive health dashboard (SCR-016) displaying my 360-Degree Patient View with demographics, conditions, medications, allergies, vital trends, and recent encounters, so that I can view my consolidated health information extracted from uploaded clinical documents with AI vs. staff-verified indicators.

## Acceptance Criteria

1. **Given** I have uploaded clinical documents that have been processed, **When** I navigate to Health Dashboard from the sidebar menu, **Then** the system displays Patient Health Dashboard page (SCR-016) with six sections: Demographics, Active Conditions, Current Medications, Known Allergies, Vital Trends Chart, and Recent Encounters.

2. **Given** I am viewing the Health Dashboard, **When** data is displayed, **Then** each extracted clinical data element shows a verification badge: amber "AI-Suggested" for unverified data or green "Staff-Verified" for data confirmed by clinical staff (UXR-402).

3. **Given** I have no uploaded documents, **When** I access Health Dashboard, **Then** the system displays empty state with illustration, message "No health data available yet. Upload your clinical documents to build your health profile.", and "Upload Documents" button that navigates to Document Upload page.

4. **Given** my Health Dashboard is loading, **When** data fetch exceeds 300ms, **Then** the system displays skeleton loaders for each section (UXR-502) and loads data progressively as API responses return.

## Edge Cases

* What happens when some sections have data but others are empty? Display populated sections normally and show section-specific empty states (e.g., "No allergies recorded" for Allergies section).
* What happens when API request for patient profile fails? Display global error banner "Unable to load health data. Please try again." with retry button and preserve skeleton loaders for failed sections.
* What happens when vital trends have insufficient data points for chart? Display message "Insufficient data for trend visualization. At least 3 entries required." in chart area.
* What happens when a clinician updates verification status while patient is viewing dashboard? Pusher real-time update changes badge from amber to green without page refresh, with subtle animation (150ms ease-out).
