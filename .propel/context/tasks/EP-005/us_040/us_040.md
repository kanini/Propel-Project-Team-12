# User Story - US_040

## Story ID

* ID: US_040

## Story Title

* Microsoft Outlook Calendar Synchronization

## Description

* As a patient, I want my booked appointments to sync with Microsoft Outlook Calendar, so that my healthcare visits integrate with my Outlook schedule.

## Acceptance Criteria

1. **Given** I have connected my Outlook Calendar, **When** I book an appointment, **Then** a calendar event is created via Microsoft Graph API with appointment date, time, provider, location, and visit reason.

2. **Given** I reschedule an appointment, **When** the new time is confirmed, **Then** the Outlook Calendar event is updated to reflect the new date and time.

3. **Given** I cancel an appointment, **When** the cancellation is confirmed, **Then** the Outlook Calendar event is removed.

4. **Given** I have not connected Outlook Calendar, **When** I view my appointment confirmation, **Then** an "Add to Outlook" button is available that initiates the Microsoft OAuth2 flow.

## Edge Cases

* What happens when Microsoft Graph API rate limits are exceeded? Requests are queued and retried after the rate limit window resets, with the user notified of the delay.
* How does the system handle users with both Google and Outlook connected? Events are created in both calendars independently.
