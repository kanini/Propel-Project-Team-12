# User Story - US_039

## Story ID

* ID: US_039

## Story Title

* Google Calendar Synchronization

## Description

* As a patient, I want my booked appointments to sync with Google Calendar, so that my healthcare visits appear alongside my other commitments.

## Acceptance Criteria

1. **Given** I have connected my Google Calendar, **When** I book an appointment, **Then** a calendar event is created via Google Calendar API with appointment date, time, provider, location, and visit reason.

2. **Given** I reschedule an appointment, **When** the new time is confirmed, **Then** the Google Calendar event is updated to reflect the new date and time.

3. **Given** I cancel an appointment, **When** the cancellation is confirmed, **Then** the Google Calendar event is removed.

4. **Given** I have not connected Google Calendar, **When** I view my appointment confirmation, **Then** an "Add to Google Calendar" button is available that initiates the OAuth2 flow.

## Edge Cases

* What happens when the Google Calendar API is temporarily unavailable? Appointment booking succeeds; calendar sync is retried asynchronously with exponential backoff.
* How does the system handle Google OAuth token expiry? Refresh tokens are used to obtain new access tokens; if refresh fails, user is prompted to re-authorize.
