# User Story - US_009

## Story ID
* ID: US_009

## Story Title
* Send Email Appointment Confirmation

## Description
* As the system, I want to automatically send appointment confirmation emails with PDF attachment and .ics calendar file, so that patients receive all necessary appointment information and calendar integration options.

## Acceptance Criteria
1. **Given** a patient successfully books an appointment and PDF generation completes, **When** the email job executes, **Then** the system sends a confirmation email within 60 seconds containing plain text appointment summary, PDF attachment, and .ics calendar file attachment via SendGrid or Mailgun free tier.

2. **Given** the email is sent, **When** the delivery status is tracked, **Then** the system records delivery status (pending/sent/failed/bounced), updates the appointment record, and logs the email event in the audit trail with timestamp and recipient.

3. **Given** email delivery fails with a transient error (network timeout, 5xx error), **When** the system detects the failure, **Then** it retries 3 times with exponential backoff (2s, 4s, 8s) and if all retries fail, logs the permanent failure and notifies admin for manual follow-up.

4. **Given** the email contains appointment details, **When** the patient opens the email, **Then** the plain text summary includes date, time, provider, location, confirmation number, reschedule link, and cancel link (both with unique security tokens).

## Edge Cases
* What happens when the patient's email address bounces (invalid/non-existent)? (System receives bounce notification, marks email status as "Bounced," flags patient record for email verification, notifies admin to contact patient via phone for correct email)
* How does the system handle emails that exceed size limits due to large PDF attachments? (PDF compression applied if file >5MB; if still exceeds, email sent without PDF attachment with download link to secure patient portal)
* What happens if SendGrid/Mailgun API rate limits are exceeded? (System detects 429 Too Many Requests, queues emails for delayed sending, staggers delivery over 60-second window to stay within free tier limits)
* How does the system ensure HIPAA compliance in email content? (Email body contains minimal PHI (patient name, appointment date only); detailed information in encrypted PDF attachment; email transmission uses TLS 1.2+; audit log records all email sends)
