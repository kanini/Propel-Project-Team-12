# User Story - US_008

## Story ID
* ID: US_008

## Story Title
* Generate PDF Appointment Confirmation

## Description
* As the system, I want to automatically generate a PDF appointment confirmation with a QR code placeholder, so that patients receive a professional, printable confirmation document for their records.

## Acceptance Criteria
1. **Given** a patient successfully books an appointment, **When** the booking transaction completes, **Then** the system triggers a background job that generates a PDF within 10 seconds containing appointment details (date, time, provider, location, confirmation number, instructions) and a QR code placeholder.

2. **Given** the PDF generation job executes, **When** the PDF is created using PdfSharpCore or Puppeteer, **Then** the PDF is formatted for both mobile and desktop viewing with readable fonts (min 12pt), clear section headers, and the QR code placeholder labeled "Future Feature: Check-In QR."

3. **Given** the PDF generation completes successfully, **When** the file is ready, **Then** the system stores the PDF in encrypted blob storage, updates the appointment record with the PDF file path, and triggers the email delivery job with the PDF attachment.

4. **Given** PDF generation fails, **When** the background job encounters an error, **Then** the system logs the error details, retries 3x with exponential backoff (2s, 4s, 8s), and if all retries fail, sends email confirmation with plain text details only and notifies admin for manual PDF generation.

## Edge Cases
* What happens when the PDF generation library crashes or times out? (System catches exception, logs stack trace with appointment ID, marks job as failed after retries exhausted, sends email without PDF, alerts admin via monitoring system)
* How does the system handle appointments with very long provider names or addresses? (PDF template uses text wrapping and dynamic font sizing to fit content; max field lengths enforced at data validation layer)
* What happens if blob storage is unavailable during PDF save? (System retries storage operation 3x, logs failure, stores PDF temporarily in local file system, queues for upload retry via background cleanup job)
* How does the system ensure HIPAA compliance for PDF storage? (Blob storage configured with AES-256 server-side encryption, access logs enabled, file paths include patient-specific folder with access restricted by patient ID)
