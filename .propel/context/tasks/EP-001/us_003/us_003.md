# User Story - US_003

## Story ID
* ID: US_003

## Story Title
* Book Appointment with Preferred Slot Swap

## Description
* As a patient, I want to book an available appointment slot while registering interest in a currently unavailable preferred slot, so that the system can automatically swap me to my preferred time if it opens up.

## Acceptance Criteria
1. **Given** I am viewing available slots, **When** I select an available slot and the system prompts "Would you prefer a different time?", **Then** I can select a currently unavailable preferred slot and the system books the available slot while registering my swap request.

2. **Given** I successfully book with a swap request, **When** the booking completes, **Then** I receive a confirmation email explaining my current booking with a note "We're monitoring your preferred slot [date/time]. You'll be notified automatically if we can swap you."

3. **Given** I have a swap request registered, **When** my preferred slot becomes available and the system executes the swap, **Then** I receive email  and SMS notifications with the new appointment details and an updated PDF confirmation.

4. **Given** I book with a swap request, **When** the preferred slot never becomes available before my appointment date, **Then** I attend my originally booked appointment and the swap request status changes to "Expired."

## Edge Cases
* What happens when I cancel my appointment before the preferred slot opens? (System cancels swap request, changes status to "Cancelled", preferred slot remains available for other patients)
* How does the system handle multiple patients requesting the same preferred slot? (FIFO processing: first patient with registered swap request gets the slot when it opens, other patients remain in their primary slots)
* What happens if I have multiple appointments with different swap requests? (Each swap request tracked independently with unique swapRequestID; one appointment can only have one active swap request)
* How does the system handle swap requests that would conflict with other appointments? (Validation check: if patient already has appointment within 2 hours of preferred slot, system rejects swap request with error "Preferred slot conflicts with existing appointment")
