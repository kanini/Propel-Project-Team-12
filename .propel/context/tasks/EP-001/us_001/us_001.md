# User Story - US_001

## Story ID
* ID: US_001

## Story Title
* Display Real-Time Appointment Slot Availability

## Description
* As a patient, I want to see real-time appointment slot availability filtered by provider, location, and appointment type, so that I can choose a convenient time for my appointment.

## Acceptance Criteria
1. **Given** I am on the appointment booking page, **When** I select a provider, location, and appointment type, **Then** the system displays only available slots within 5 seconds with timezone information clearly shown.

2. **Given** available slots are displayed, **When** another patient books a slot while I am viewing, **Then** the booked slot disappears from my view within 5 seconds without requiring a page refresh.

3. **Given** I am viewing appointment slots, **When** I filter by a different provider or location, **Then** the system updates the available slot list within 5 seconds showing only slots matching my new criteria.

4. **Given** no slots are available for my selected criteria, **When** the system completes the search, **Then** I see a message "No available slots found. Try different criteria or join waitlist" with alternative suggestions.

## Edge Cases
* What happens when the system detects a booking conflict while displaying slots (two patients viewing the same slot)? (Optimistic locking: first to book wins, second sees "Slot No Longer Available" with alternative suggestions)
* How does the system handle timezone display for multi-location practices? (Each slot displays timezone abbreviation like "EST" or "PST"; patient's browser timezone used for default view)
* What happens when the API request times out while loading slots? (Display error message "Unable to load slots. Please try again" with retry button; system logs timeout for monitoring)
* How does the system handle slots that become available during viewing? (New slots appear automatically within 5 seconds via polling or WebSocket refresh; displayed with subtle highlight animation)
