# User Story - US_067

## Story ID

* ID: US_067

## Story Title

* Patient Dashboard - Post-Login Landing Page

## Description

* As a patient, I want a personalized dashboard as my post-login landing page showing upcoming appointments, health summary stats, quick action cards, and recent notifications, so that I can quickly access key features and stay informed about my health status.

## Acceptance Criteria

1. **Given** I am a patient who has just logged in, **When** authentication completes successfully (UC-002), **Then** the system redirects me to the Patient Dashboard (SCR-003) displaying my personalized content within 500ms (NFR-001).

2. **Given** I am viewing the Patient Dashboard, **When** the page renders, **Then** the system displays three statistical cards showing: total appointments (past 6 months), upcoming appointments (next 30 days), and waitlist entries (current count) with appropriate trend indicators per UXR-502.

3. **Given** I am viewing the dashboard, **When** the page loads, **Then** the system displays three quick action cards: "Book Appointment" (navigates to SCR-006), "Complete Intake" (navigates to SCR-012), and "Upload Documents" (navigates to SCR-014), meeting UXR-001 (max 3 clicks to any feature).

4. **Given** I have upcoming appointments, **When** the dashboard renders, **Then** the system displays my next 5 upcoming appointments in chronological order showing provider name, date/time, specialty, and appointment status badge (Confirmed/Waitlist/Pending) per UXR-401.

5. **Given** I have notifications (slot swap available, appointment reminders, document processing complete), **When** the dashboard loads, **Then** the system displays a notification panel showing the 5 most recent unread notifications with timestamps and action links.

6. **Given** I have recently uploaded clinical documents, **When** I view the dashboard, **Then** the system displays a "Recent Documents" section showing the 3 most recent documents with processing status badges (Processing/Completed/Failed) per UXR-402.

7. **Given** I click any of the quick action cards or navigation items, **When** the action executes, **Then** the system provides visual feedback within 200ms (button state change) and navigates to the target screen per UXR-501.

8. **Given** the dashboard is loading data, **When** API response time exceeds 300ms, **Then** the system displays skeleton loading states for stat cards, appointment list, and notification panel per UXR-502.

9. **Given** I am a new patient with no data, **When** the dashboard loads, **Then** the system displays empty state illustrations with guiding CTAs per UXR-605: "No appointments yet. Browse providers to schedule your first visit."

10. **Given** I access the dashboard on mobile viewport (320px-767px), **When** the page renders, **Then** the system adapts layout to single column stacking stat cards and quick actions per UXR-301 and UXR-303.

## Edge Cases

* What happens when appointment data fetch fails? Display error banner with retry action per UXR-603 while showing empty placeholder sections.
* How does the system handle 50+ notifications? Display most recent 5 with "View All Notifications" link to dedicated notifications page.
* What happens when a patient has both regular appointments and waitlist entries? Display appointments sorted chronologically with waitlist badge distinction; waitlist entries shown separately in "My Appointments" tab.
* How does the dashboard handle real-time slot swap notifications? Display modal overlay per figma_spec.md "Slot Swap Notification" modal design; update dashboard state after user confirms/declines.
