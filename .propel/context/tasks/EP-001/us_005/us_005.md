# User Story - US_005

## Story ID
* ID: US_005

## Story Title
* Join Waitlist for Fully Booked Schedules

## Description
* As a patient, I want to join a waitlist when all appointment slots are fully booked, so that I can be notified when a slot becomes available and have priority booking access.

## Acceptance Criteria
1. **Given** I am viewing appointment availability, **When** all slots for my desired date are fully booked, **Then** the system displays a "Join Waitlist" button and explains "You'll be notified when a slot opens. You'll have 2 hours to book."

2. **Given** I join the waitlist, **When** my request is processed, **Then** the system adds me to the FIFO waitlist queue, sends confirmation email "You're on the waitlist for [date]. We'll notify you when a slot opens," and displays my queue position.

3. **Given** I am on the waitlist, **When** a slot becomes available, **Then** the system sends me email and SMS notification with booking link valid for 2 hours and updates my waitlist status to "Notified."

4. **Given** I receive a waitlist notification, **When** I do not respond within 2 hours, **Then** the system marks my response as "No Response," moves me to the back of the waitlist, and notifies the next patient in queue.

## Edge Cases
* What happens when multiple slots open simultaneously with multiple waitlisted patients? (System notifies patients in FIFO order sequentially; each patient gets 2-hour window; if first patient books, second patient notified; if both decline, slots released to general availability)
* How does the system handle waitlist requests that conflict with my existing appointments? (Validation check: if patient already has appointment within 2 hours of requested waitlist date, system rejects request with error "Waitlist date conflicts with existing appointment")
* What happens if I'm on multiple waitlists for different dates? (Each waitlist entry tracked independently; patient can be on multiple waitlists simultaneously; notification sent for each slot that opens)
* How does the system handle waitlist notification delivery failures? (System retries notification 3x with exponential backoff; if all fail, logs error and moves to next waitlisted patient to avoid blocking queue; failed notification logged for admin review)
