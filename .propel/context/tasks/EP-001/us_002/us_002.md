# User Story - US_002

## Story ID
* ID: US_002

## Story Title
* Book Available Appointment Slot

## Description
* As a patient, I want to book an available appointment slot with single-click confirmation, so that I can quickly secure my appointment and receive immediate confirmation.

## Acceptance Criteria
1. **Given** I am viewing available slots, **When** I click on a slot and confirm booking, **Then** the system creates the appointment, displays a confirmation number, and returns a success message within 3 seconds.

2. **Given** I successfully book an appointment, **When** the booking completes, **Then** I receive a confirmation email with PDF attachment within 60 seconds containing appointment details (date, time, provider, location, confirmation number).

3. **Given** I attempt to book a slot, **When** another patient books the same slot simultaneously, **Then** the system displays "Slot No Longer Available" error with 409 Conflict status and suggests 3 alternative available slots.

4. **Given** I have booked an appointment, **When** I navigate to my patient dashboard, **Then** the new appointment appears in my upcoming appointments list with status "Scheduled" and all booking details.

## Edge Cases
* What happens when PDF generation fails during booking? (System still confirms booking, sends email with plain text details only, logs error, notifies admin for manual PDF generation)
* How does the system handle booking when the patient has incomplete intake data? (Booking proceeds successfully; system displays reminder "Complete intake form before appointment" with link to intake form)
* What happens if the confirmation email fails to send? (System logs error, displays warning "Email delivery pending. Check spam folder or contact support if not received within 5 minutes", retries 3x with exponential backoff)
* How does the system handle concurrent booking conflicts with optimistic locking? (RowVersion check during transaction; first commit wins, second transaction fails with DbUpdateConcurrencyException, user sees conflict error)
