# User Story - US_022

## Story ID

* ID: US_022

## Story Title

* Auth Audit Logging and Session Timeout Warning

## Description

* As a platform operator, I want immutable audit logs for all authentication events and a session timeout warning modal, so that security events are tracked and users are warned before automatic logout.

## Acceptance Criteria

1. **Given** a user logs in successfully, **When** the authentication event occurs, **Then** an immutable audit log entry is created with user ID, timestamp, action type (login), IP address, and user agent.

2. **Given** a login attempt fails, **When** invalid credentials are submitted, **Then** an audit log entry records the failed attempt with email (hashed), timestamp, IP, and failure reason.

3. **Given** a session timeout occurs, **When** the 15-minute TTL expires, **Then** an audit log entry records the session timeout with user ID and last activity timestamp.

4. **Given** a user has been active for 13 minutes, **When** the session approaches timeout, **Then** a warning modal appears with a countdown timer and an "Extend Session" button that refreshes the TTL.

5. **Given** the warning modal is displayed, **When** the user clicks "Extend Session", **Then** the session TTL resets to 15 minutes and the modal closes; if ignored, the session expires at 15 minutes and user is logged out.

## Edge Cases

* What happens when the audit log service is temporarily unavailable? Auth events should still succeed (audit is non-blocking) but failures are logged to application error logs for retry.
* How does the system handle the timeout warning when the user has multiple browser tabs open? Session extension in any tab should dismiss the warning in all tabs via shared session state.
