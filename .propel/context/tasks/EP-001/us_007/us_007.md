# User Story - US_007

## Story ID
* ID: US_007

## Story Title
* Sync Appointments to Outlook Calendar

## Description
* As a patient, I want my booked appointments automatically synced to my Outlook Calendar, so that I can manage my appointments within the Microsoft ecosystem without manual data entry.

## Acceptance Criteria
1. **Given** I am booking an appointment, **When** I authorize Outlook Calendar access via Microsoft OAuth 2.0, **Then** the system creates a calendar event using Microsoft Graph API with appointment details (date, time, provider, location, instructions) and confirms "Synced to Outlook Calendar."

2. **Given** my appointment is synced to Outlook Calendar, **When** I reschedule or cancel the appointment, **Then** the system automatically updates or deletes the Outlook Calendar event via Microsoft Graph API /me/events endpoint within 60 seconds and displays "Calendar updated."

3. **Given** the system attempts to sync to Outlook Calendar, **When** the API returns a 5xx server error, **Then** the system retries with exponential backoff (1s, 2s, 4s) up to 3 times and falls back to including .ics file in email if all retries fail.

4. **Given** 5 consecutive calendar sync failures occur, **When** the circuit breaker opens, **Then** the system stops attempting Outlook Calendar sync for 60 seconds, logs the circuit breaker event, displays "Calendar sync temporarily unavailable. Use .ics file attachment," and retries after cooldown.

## Edge Cases
* What happens when I revoke Outlook Calendar access after appointments are synced? (System detects auth failure on next sync attempt, displays warning "Calendar access revoked. Re-authorize to continue syncing," keeps appointments in local database, stops sync attempts until re-authorization)
* How does the system handle appointments booked before calendar authorization? (Once authorized, system displays "Sync past appointments to calendar?" with option to bulk-sync historical appointments or sync only future appointments)
* What happens if Microsoft Graph API token expires during sync? (System detects 401 Unauthorized, attempts token refresh with refresh_token, retries sync; if refresh fails, prompts patient to re-authorize)
* How does the system handle timezone mismatches between system and Outlook Calendar? (System includes timezone information in calendar event creation via Microsoft Graph API; event displays correctly in patient's local timezone in Outlook Calendar)
