# User Story - US_021

## Story ID

* ID: US_021

## Story Title

* Admin User Management

## Description

* As an Admin, I want to create, update, and deactivate Staff and Admin user accounts, so that I can control platform access and manage the workforce effectively.

## Acceptance Criteria

1. **Given** I am on the User Management page, **When** I click "Create User", **Then** a form displays for entering user name, email, role (Staff/Admin), and the system sends an activation email upon save.

2. **Given** I am editing a user account, **When** I modify user details (name, role) and save, **Then** the changes are persisted, an audit log entry is created, and the update is confirmed.

3. **Given** I am deactivating a user, **When** I click "Deactivate" and confirm, **Then** the account status changes to inactive, all active sessions for that user are terminated, and future login attempts are blocked.

4. **Given** I am viewing the user list, **When** the page loads, **Then** all Staff and Admin users are displayed in a table with name, email, role, status, and last login columns with sorting and filtering.

5. **Given** an Admin is managing users, **When** they attempt to deactivate their own account, **Then** the system prevents self-deactivation with an error message.

## Edge Cases

* What happens when creating a user with an email already in use? System displays "Email already registered" error and prevents duplicate creation.
* How does the system handle deactivation of the last Admin account? System prevents deactivation if it would leave zero active Admin accounts.
