# User Story - US_069

## Story ID

* ID: US_069

## Story Title

* Admin Dashboard - System Management Hub

## Description

* As an admin, I want a dashboard as my post-login landing page showing system statistics (total users, active sessions, recent actions), audit log highlights, and quick access to administrative functions, so that I can monitor system health and efficiently perform administrative tasks.

## Acceptance Criteria

1. **Given** I am an admin who has just logged in, **When** authentication completes successfully (UC-002), **Then** the system redirects me to the Admin Dashboard (SCR-005) displaying system metrics within 500ms (NFR-001).

2. **Given** I am viewing the Admin Dashboard, **When** the page renders, **Then** the system displays four statistical cards showing: total active users (by role count), active sessions (current online users), recent registrations (last 7 days), and pending user approvals (if any) with appropriate trend indicators.

3. **Given** I am viewing the dashboard, **When** the page loads, **Then** the system displays three quick action cards: "User Management" (navigates to SCR-021), "View Audit Log" (navigates to SCR-025), and "System Settings" (navigates to SCR-026), meeting UXR-001 (max 3 clicks to any feature).

4. **Given** there are recent audit events, **When** the dashboard renders, **Then** the system displays a "Recent Audit Events" widget showing the 10 most recent administrative actions (user created, role changed, account deactivated, access denied) with timestamp, actor, action type, and target user per FR-005.

5. **Given** there are active user sessions, **When** the dashboard loads, **Then** the system displays a "Active Sessions" widget showing current online users with username, role, login timestamp, and last activity time, allowing identification of stale sessions.

6. **Given** there are failed authentication attempts, **When** the dashboard renders, **Then** the system displays a "Security Alerts" widget showing failed login attempts with username, timestamp, IP address (if available), and lockout status if threshold exceeded (5+ failures).

7. **Given** I click any of the quick action cards or audit event items, **When** the action executes, **Then** the system provides visual feedback within 200ms and navigates to the target screen per UXR-501.

8. **Given** the dashboard is loading data, **When** API response time exceeds 300ms, **Then** the system displays skeleton loading states for stat cards, audit log widget, and sessions panel per UXR-502.

9. **Given** there are no recent administrative actions, **When** the dashboard loads, **Then** the system displays empty state illustration per UXR-605: "No recent activity. System operating normally."

10. **Given** I access the dashboard on mobile viewport (320px-767px), **When** the page renders, **Then** the system adapts layout to single column stacking stat cards and widgets per UXR-301 and UXR-303.

## Edge Cases

* What happens when audit log fetch fails? Display error banner with retry action per UXR-603 while showing empty placeholder for audit widget.
* How does the system handle 200+ active sessions? Display count in stat card; widget shows 10 most recent with "View All Sessions" link (future feature).
* What happens when critical security events occur (repeated failed logins, deactivated user attempting access)? Display security alert banner at top of dashboard with red warning icon and "Review Audit Log" action.
* How does the dashboard differentiate between routine and critical audit events? Color-coded badges: Info (blue), Warning (amber), Critical (red) for security-related events.
