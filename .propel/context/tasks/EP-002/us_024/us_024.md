# User Story - US_024

## Story ID

* ID: US_024

## Story Title

* Availability Calendar and Appointment Booking

## Description

* As a patient, I want to view real-time availability in a calendar and book an appointment by selecting provider, date, time, and visit reason, so that I can schedule my healthcare visit efficiently.

## Acceptance Criteria

1. **Given** I have selected a provider, **When** the availability page loads, **Then** a calendar view displays with available time slots highlighted and unavailable slots grayed out for the selected provider.

2. **Given** I am viewing the calendar, **When** I select a date, **Then** available time slots for that date are displayed with duration and the API responds within 500ms at the 95th percentile (NFR-001).

3. **Given** I have selected a time slot, **When** I enter my visit reason and click "Book Appointment", **Then** the system validates slot availability (FOR UPDATE lock to prevent double-booking), creates the appointment record, and displays a confirmation screen.

4. **Given** another patient books the same slot concurrently, **When** I attempt to book, **Then** the system detects the conflict (409 Conflict), displays "Slot no longer available", and refreshes the calendar to show updated availability.

5. **Given** I am in the multi-step booking flow, **When** I am on any step, **Then** a progress indicator shows the current step (Provider → Date/Time → Details → Confirm) per UXR-101.

## Edge Cases

* What happens when I navigate months ahead and no slots exist? Calendar shows "No availability" for selected month with option to join waitlist.
* How does the system handle slow API responses for availability? Skeleton loading state appears after 300ms and an error message with retry is shown after 10-second timeout.
