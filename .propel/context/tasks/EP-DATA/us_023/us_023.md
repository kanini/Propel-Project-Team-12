# User Story - US_023

## Story ID
* ID: US_023

## Story Title
* Encrypted Blob Storage Configuration

## Description
* As a backend developer, I want encrypted blob storage configured for clinical PDF uploads with server-side AES-256 encryption, so that patient documents are protected at rest and meet HIPAA encryption requirements while staying within free-tier limits.

## Acceptance Criteria
1. **Given** the backend API exists, **When** I configure blob storage, **Then** I choose one of:
   - **Option A**: Supabase Storage (free tier: 1GB storage, unlimited downloads)
   - **Option B**: Cloudflare R2 (free tier: 10GB/month, 1M writes)
   - Both support server-side encryption with customer-provided keys

2. **Given** storage is configured, **When** a PDF is uploaded, **Then** the system:
   - Generates unique filename: `{PatientID}/{DocumentID}_{Timestamp}.pdf`
   - Encrypts file server-side with AES-256 before storage
   - Stores encrypted file path in `ClinicalDocument.FilePath` column
   - Returns signed URL with 1-hour expiration for secure download

3. **Given** the encryption key is managed, **When** I review configuration, **Then** I see:
   - Encryption master key stored in environment variable: `BLOB_STORAGE_ENCRYPTION_KEY`
   - Key rotation procedure documented (US_024)
   - Key never logged or exposed in API responses

4. **Given** download requests occur, **When** staff accesses a clinical document, **Then**:
   - API generates temporary signed URL (1-hour expiration)
   - URL includes authentication token to prevent unauthorized access
   - Downloaded file is decrypted client-side by browser (automatic for HTTPS)

## Edge Cases
* What happens when encryption key is rotated? (New uploads use new key; old files remain encrypted with old key; dual-key decryption support for transition period)
* How does system handle file size exceeding 10MB limit? (API rejects upload with error: "File size exceeds 10MB limit. Please compress or split into multiple files.")

## Technical Notes
- **Option A: Supabase Storage Configuration**
  ```csharp
  // Install: dotnet add package supabase-csharp
  public class SupabaseBlobStorage : IBlobStorageService
  {
      private readonly Supabase.Client _client;

      public SupabaseBlobStorage(IConfiguration config)
      {
          var url = config["Supabase:Url"];
          var key = config["Supabase:Key"];
          _client = new Supabase.Client(url, key);
      }

      public async Task<string> UploadDocumentAsync(Stream fileStream, string fileName)
      {
          var bucket = "clinical-documents";
          var path = $"{Guid.NewGuid()}/{fileName}";

          await _client.Storage.From(bucket).Upload(fileStream, path);

          // Supabase provides server-side encryption by default
          return path; // Store this path in ClinicalDocument.FilePath
      }

      public async Task<string> GetSignedUrlAsync(string filePath, int expirationSeconds = 3600)
      {
          var bucket = "clinical-documents";
          var signedUrl = await _client.Storage.From(bucket).CreateSignedUrl(filePath, expirationSeconds);
          return signedUrl;
      }
  }
  ```
- **Option B: Cloudflare R2 Configuration**
  ```csharp
  // Install: dotnet add package AWSSDK.S3 (R2 is S3-compatible)
  public class CloudflareR2BlobStorage : IBlobStorageService
  {
      private readonly AmazonS3Client _client;

      public CloudflareR2BlobStorage(IConfiguration config)
      {
          var accessKey = config["R2:AccessKeyId"];
          var secretKey = config["R2:SecretAccessKey"];
          var endpoint = config["R2:Endpoint"]; // https://<account-id>.r2.cloudflarestorage.com

          var s3Config = new AmazonS3Config
          {
              ServiceURL = endpoint,
              ForcePathStyle = true
          };

          _client = new AmazonS3Client(accessKey, secretKey, s3Config);
      }

      public async Task<string> UploadDocumentAsync(Stream fileStream, string fileName)
      {
          var bucketName = "clinical-documents";
          var key = $"{Guid.NewGuid()}/{fileName}";

          var request = new PutObjectRequest
          {
              BucketName = bucketName,
              Key = key,
              InputStream = fileStream,
              ServerSideEncryptionMethod = ServerSideEncryptionMethod.AES256
          };

          await _client.PutObjectAsync(request);
          return key;
      }

      public async Task<string> GetSignedUrlAsync(string filePath, int expirationMinutes = 60)
      {
          var request = new GetPreSignedUrlRequest
          {
              BucketName = "clinical-documents",
              Key = filePath,
              Expires = DateTime.UtcNow.AddMinutes(expirationMinutes)
          };

          return _client.GetPreSignedURL(request);
      }
  }
  ```

- API controller for upload:
  ```csharp
  [HttpPost("documents/upload")]
  [Authorize(Roles = "Patient,Staff")]
  public async Task<IActionResult> UploadDocument(IFormFile file)
  {
      if (file.Length > 10 * 1024 * 1024) // 10MB
      {
          return BadRequest("File size exceeds 10MB limit.");
      }

      using var stream = file.OpenReadStream();
      var filePath = await _blobStorage.UploadDocumentAsync(stream, file.FileName);

      var document = new ClinicalDocument
      {
          DocumentID = Guid.NewGuid(),
          PatientID = GetCurrentUserId(),
          FilePath = filePath,
          FileSize = file.Length,
          UploadedAt = DateTime.UtcNow,
          ProcessingStatus = ProcessingStatus.Pending
      };

      _context.ClinicalDocuments.Add(document);
      await _context.SaveChangesAsync();

      return Ok(document);
  }
  ```
- Security considerations:
  - Never expose direct blob storage URLs to clients (use signed URLs)
  - Validate file types (PDF only in Phase 1) before upload
  - Scan files for viruses via VirusTotal API (free tier: 4 requests/minute)
  - Implement rate limiting on uploads to prevent abuse
