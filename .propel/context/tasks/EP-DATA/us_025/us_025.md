# User Story - US_025

## Story ID
* ID: US_025

## Story Title
* Mock Insurance Database Seed

## Description
* As a backend developer, I want a seeded mock insurance database with 50 common payers imported from CSV, so that the system can perform soft validation of patient insurance information during pre-check workflows without integrating with expensive real-time verification APIs.

## Acceptance Criteria
1. **Given** the InsuranceProviders table schema is defined, **When** I review the entity model, **Then** it includes:
   - ProviderID (GUID primary key)
   - ProviderName (string, e.g., "Blue Cross Blue Shield")
   - PayerID (string, e.g., "BCBS-CA")
   - IsActive (bool, default true)
   - SupportedStates (string array, e.g., ["CA", "NY", "TX"])

2. **Given** the CSV seed data file exists, **When** I inspect it, **Then** it contains:
   - 50 rows with common payers: Blue Cross, Aetna, UnitedHealthcare, Cigna, Humana, Medicare, Medicaid (state-specific), Kaiser Permanente, etc.
   - Columns: ProviderName, PayerID, SupportedStates (pipe-delimited: "CA|NY|TX")

3. **Given** the seed migration is created, **When** I apply it, **Then** the database:
   - Reads CSV file from `Infrastructure/Data/Seeds/insurance_providers.csv`
   - Parses rows and inserts into InsuranceProviders table
   - Logs insertion count: "Seeded 50 insurance providers"

4. **Given** insurance validation is implemented, **When** a patient enters insurance info during booking, **Then** the API:
   - Queries: `SELECT * FROM InsuranceProviders WHERE ProviderName ILIKE '%blue cross%'`
   - Returns matched providers as suggestions (typeahead search)
   - Soft validation: Warns if provider not found, but allows booking to proceed (FR-021)

## Edge Cases
* What happens when CSV file is missing or malformed? (Seed migration logs warning and skips seed; database remains functional; admin manually adds providers via UI in Phase 2)
* How does system handle updates to insurance provider list? (Manual edit via SQL or admin UI; auto-seed only runs on initial migration; update mechanism documented for future phases)

- Seed migration:
  ```csharp
  public partial class SeedInsuranceProviders : Migration
  {
      protected override void Up(MigrationBuilder migrationBuilder)
      {
          var csvPath = Path.Combine(AppContext.BaseDirectory, "Infrastructure", "Data", "Seeds", "insurance_providers.csv");

          if (!File.Exists(csvPath))
          {
              Console.WriteLine($"WARNING: CSV file not found at {csvPath}. Skipping insurance provider seed.");
              return;
          }

          var lines = File.ReadAllLines(csvPath);
          var insertCount = 0;

          foreach (var line in lines.Skip(1)) // Skip header
          {
              var parts = line.Split(',');
              if (parts.Length != 3) continue;

              var providerName = parts[0];
              var payerID = parts[1];
              var supportedStates = parts[2].Split('|');

              migrationBuilder.InsertData(
                  table: "InsuranceProviders",
                  columns: new[] { "ProviderID", "ProviderName", "PayerID", "IsActive", "SupportedStates" },
                  values: new object[] { Guid.NewGuid(), providerName, payerID, true, supportedStates });

              insertCount++;
          }

          Console.WriteLine($"Seeded {insertCount} insurance providers.");
      }

      protected override void Down(MigrationBuilder migrationBuilder)
      {
          migrationBuilder.DeleteData(table: "InsuranceProviders");
      }
  }
  ```

- Typeahead search API:
  ```csharp
  [HttpGet("insurance/search")]
public async Task<IActionResult> SearchInsuranceProviders([FromQuery] string query)
  {
      var providers = await _context.InsuranceProviders
          .Where(p => EF.Functions.ILike(p.ProviderName, $"%{query}%"))
          .OrderBy(p => p.ProviderName)
          .Take(10)
          .ToListAsync();

      return Ok(providers);
  }
  ```

- Soft validation logic:
  ```csharp
  public async Task<ValidationResult> ValidateInsurance(string providerName, string memberId)
  {
      var provider = await _context.InsuranceProviders
          .FirstOrDefaultAsync(p => p.ProviderName.ToLower().Contains(providerName.ToLower()));

      if (provider == null)
      {
          return ValidationResult.Warning("Insurance provider not found in our system. You may still proceed with booking.");
      }

      // Phase 2: Real-time verification via external API
      return ValidationResult.Success();
  }
  ```
