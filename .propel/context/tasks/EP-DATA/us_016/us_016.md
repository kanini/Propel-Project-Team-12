# User Story - US_016

## Story ID
* ID: US_016

## Story Title
* Clinical Data Entity Models (ClinicalDocument, ExtractedData)

## Description
* As a backend developer, I want ClinicalDocument and ExtractedData entities with pgvector embeddings and confidence scoring, so that AI-powered extraction and retrieval workflows can store and query clinical data with traceability to source documents.

## Acceptance Criteria
1. **Given** the Domain layer exists, **When** I create entity classes, **Then** I have:
   - `ClinicalDocument.cs` with properties: DocumentID (GUID), PatientID (FK), DocumentType (enum: Lab/Imaging/Referral/PostVisitNote), UploadedAt, UploadedByUserID (FK), FilePath (encrypted blob URL string), FileSize (bytes), EmbeddingVector (pgvector column, 1536 dimensions), ProcessingStatus (enum: Pending/Processing/Completed/Failed)
   - `ExtractedData.cs` with properties: ExtractedDataID (GUID), DocumentID (FK), DataType (enum: Vital/Medication/Diagnosis/Allergy/Procedure/LabValue), Value (JSON string), ConfidenceScore (decimal 0.0-1.0), ExtractedAt, VerifiedByStaffID (FK nullable), VerifiedAt (nullable), SourceDocumentPageNumber (int nullable)

2. **Given** the entities use Fluent API configuration, **When** I configure `ClinicalDocument`, **Then** it includes:
   - pgvector column: `.Property(e => e.EmbeddingVector).HasColumnType("vector(1536)")`
   - Foreign key to Patient with cascade delete: `.HasOne(d => d.Patient).WithMany(p => p.ClinicalDocuments).OnDelete(DeleteBehavior.Cascade)`
   - FilePath encryption handled at application layer (not database-level encryption)

3. **Given** ExtractedData is configured, **When** I inspect the schema, **Then** I see:
   - ConfidenceScore with CHECK constraint: `ConfidenceScore >= 0.0 AND ConfidenceScore <= 1.0`
   - Index on ConfidenceScore for low-confidence query: `.HasIndex(e => e.ConfidenceScore).HasFilter("ConfidenceScore < 0.7")`
   - Foreign key to ClinicalDocument: `.HasOne(e => e.ClinicalDocument).WithMany(d => d.ExtractedData).HasForeignKey(e => e.DocumentID)`

4. **Given** ConsolidatedPatientView is needed, **When** I create materialized view query, **Then** it aggregates:
   - Patient medications, allergies, diagnoses from ExtractedData grouped by DataType
   - Conflict flags where multiple documents have contradictory values (e.g., "No medications" vs "Aspirin")
   - Source document references for traceability

## Edge Cases
* What happens when embedding dimension changes in future AI model versions? (Schema migration adds new column; legacy data re-embedded via background job)
* How does system handle JSON schema evolution in Value field? (JSON column is schemaless; application validates deserialization; future: add JSON schema constraints)

## Technical Notes
- ClinicalDocument entity:
  ```csharp
  public class ClinicalDocument
  {
      public Guid DocumentID { get; set; }
      public Guid PatientID { get; set; }
      public DocumentType DocumentType { get; set; }
      public DateTime UploadedAt { get; set; }
      public Guid UploadedByUserID { get; set; }
      public string FilePath { get; set; } = string.Empty; // Encrypted blob URL
      public long FileSize { get; set; } // Bytes
      public float[]? EmbeddingVector { get; set; } // pgvector 1536 dimensions
      public ProcessingStatus ProcessingStatus { get; set; }
      
      // Navigation properties
      public Patient Patient { get; set; } = null!;
      public ICollection<ExtractedData> ExtractedDataItems { get; set; } = new List<ExtractedData>();
  }

  public enum DocumentType { Lab, Imaging, Referral, PostVisitNote }
  public enum ProcessingStatus { Pending, Processing, Completed, Failed }
  ```
- ExtractedData entity:
  ```csharp
  public class ExtractedData
  {
      public Guid ExtractedDataID { get; set; }
      public Guid DocumentID { get; set; }
      public DataType DataType { get; set; }
      public string Value { get; set; } = "{}"; // JSON string
      public decimal ConfidenceScore { get; set; }
      public DateTime ExtractedAt { get; set; }
      public Guid? VerifiedByStaffID { get; set; }
      public DateTime? VerifiedAt { get; set; }
      public int? SourceDocumentPageNumber { get; set; }
      
      // Navigation properties
      public ClinicalDocument ClinicalDocument { get; set; } = null!;
      public Staff? VerifiedByStaff { get; set; }
  }

  public enum DataType { Vital, Medication, Diagnosis, Allergy, Procedure, LabValue }
  ```
- Fluent API for pgvector:
  ```csharp
  modelBuilder.Entity<ClinicalDocument>(entity =>
  {
      entity.Property(e => e.EmbeddingVector).HasColumnType("vector(1536)");
      // Note: Requires pgvector extension enabled (US_022)
  });

  modelBuilder.Entity<ExtractedData>(entity =>
  {
      entity.Property(e => e.ConfidenceScore).HasPrecision(3, 2); // 0.00 to 1.00
      entity.HasCheckConstraint("CK_ExtractedData_ConfidenceScore", "ConfidenceScore >= 0.0 AND ConfidenceScore <= 1.0");
      entity.HasIndex(e => e.ConfidenceScore).HasFilter("ConfidenceScore < 0.7"); // Low-confidence index
  });
  ```
- ConsolidatedPatientView (materialized view or denormalized table):
  ```sql
  CREATE MATERIALIZED VIEW ConsolidatedPatientView AS
  SELECT 
      p.PatientID,
      jsonb_agg(DISTINCT ed.Value) FILTER (WHERE ed.DataType = 'Medication') AS Medications,
      jsonb_agg(DISTINCT ed.Value) FILTER (WHERE ed.DataType = 'Allergy') AS Allergies,
      jsonb_agg(DISTINCT ed.Value) FILTER (WHERE ed.DataType = 'Diagnosis') AS Diagnoses
  FROM Patients p
  LEFT JOIN ClinicalDocuments cd ON p.PatientID = cd.PatientID
  LEFT JOIN ExtractedData ed ON cd.DocumentID = ed.DocumentID
  GROUP BY p.PatientID;
  ```
