# User Story - US_018

## Story ID
* ID: US_018

## Story Title
* Immutable Audit Log Entity

## Description
* As a backend developer, I want an AuditLog entity with append-only semantics to record all CRUD operations on PHI, so that the system maintains HIPAA-compliant audit trails with 7-year retention for compliance reporting.

## Acceptance Criteria
1. **Given** the Domain layer exists, **When** I create the AuditLog entity, **Then** it includes properties:
   - AuditLogID (GUID primary key)
   - UserID (GUID FK to polymorphic user: Patient/Staff/Admin)
   - UserRole (enum: Patient/Staff/Admin)
   - Action (enum: Create/Read/Update/Delete/Login/Logout/AIInference)
   - EntityType (string, e.g., "Patient", "Appointment", "ClinicalDocument")
   - EntityID (GUID of affected entity)
   - Timestamp (DateTime with precision to milliseconds)
   - IPAddress (string)
   - UserAgent (string)
   - ChangeDetails (JSON string with before/after values for updates)

2. **Given** AuditLog is configured with Fluent API, **When** I review the schema, **Then** I see:
   - No foreign key constraints (to prevent delete cascades affecting audit trail)
   - Index on Timestamp for temporal queries: `.HasIndex(e => e.Timestamp)`
   - Index on UserID for user activity reports: `.HasIndex(e => e.UserID)`
   - Index on EntityType and EntityID for entity-specific audit queries: `.HasIndex(e => new { e.EntityType, e.EntityID })`

3. **Given** the table is created, **When** I attempt UPDATE or DELETE on AuditLog rows via SQL, **Then** the database trigger (from US_021) blocks the operation and raises an exception.

4. **Given** audit logging is operational, **When** I query logs for compliance reports, **Then** I can filter by:
   - User activity: `WHERE UserID = @userId AND Timestamp BETWEEN @start AND @end`
   - Entity changes: `WHERE EntityType = 'Patient' AND Action = 'Update'`
   - Access attempts: `WHERE Action = 'Read' AND EntityType = 'ClinicalDocument'`

## Edge Cases
* What happens when audit log table grows beyond storage limits? (Partition table by year; archive logs older than 7 years to cold storage; documented retention policy in US_024)
* How does system handle audit logging failures (e.g., database connection lost)? (Audit writes are asynchronous via message queue; failures logged to file system as fallback)

## Technical Notes
- AuditLog entity:
  ```csharp
  public class AuditLog
  {
      public Guid AuditLogID { get; set; }
      public Guid UserID { get; set; } // Polymorphic: Patient/Staff/Admin
      public UserRole UserRole { get; set; }
      public AuditAction Action { get; set; }
      public string EntityType { get; set; } = string.Empty;
      public Guid EntityID { get; set; }
      public DateTime Timestamp { get; set; }
      public string IPAddress { get; set; } = string.Empty;
      public string UserAgent { get; set; } = string.Empty;
      public string ChangeDetails { get; set; } = "{}"; // JSON

      // No navigation properties (immutable audit trail)
  }

  public enum AuditAction { Create, Read, Update, Delete, Login, Logout, AIInference }
  ```
- Fluent API configuration:
  ```csharp
  modelBuilder.Entity<AuditLog>(entity =>
  {
      entity.HasKey(e => e.AuditLogID);
      entity.Property(e => e.Timestamp).HasDefaultValueSql("NOW()");
      entity.HasIndex(e => e.Timestamp);
      entity.HasIndex(e => e.UserID);
      entity.HasIndex(e => new { e.EntityType, e.EntityID });
      entity.ToTable("AuditLogs");
      
      // No foreign key constraints to avoid cascade deletes
      entity.HasNoKey(); // Alternative: Keep key but no FK relationships
  });
  ```
- SaveChanges interceptor to auto-log CRUD operations:
  ```csharp
  public class AuditLogInterceptor : SaveChangesInterceptor
  {
      private readonly IHttpContextAccessor _httpContextAccessor;

      public override async ValueTask<InterceptionResult<int>> SavingChangesAsync(
          DbContextEventData eventData,
          InterceptionResult<int> result,
          CancellationToken cancellationToken = default)
      {
          var context = eventData.Context;
          if (context == null) return result;

          var entries = context.ChangeTracker.Entries()
              .Where(e => e.State == EntityState.Added || 
                         e.State == EntityState.Modified || 
                         e.State == EntityState.Deleted);

          foreach (var entry in entries)
          {
              var auditLog = new AuditLog
              {
                  AuditLogID = Guid.NewGuid(),
                  UserID = GetCurrentUserId(),
                  UserRole = GetCurrentUserRole(),
                  Action = entry.State == EntityState.Added ? AuditAction.Create :
                           entry.State == EntityState.Modified ? AuditAction.Update :
                           AuditAction.Delete,
                  EntityType = entry.Entity.GetType().Name,
                  EntityID = GetEntityId(entry),
                  Timestamp = DateTime.UtcNow,
                  IPAddress = _httpContextAccessor.HttpContext?.Connection.RemoteIpAddress?.ToString() ?? "Unknown",
                  UserAgent = _httpContextAccessor.HttpContext?.Request.Headers["User-Agent"].ToString() ?? "Unknown",
                  ChangeDetails = SerializeChanges(entry)
              };

              context.Set<AuditLog>().Add(auditLog);
          }

          return result;
      }
  }
  ```
- Register interceptor in Program.cs:
  ```csharp
  builder.Services.AddDbContext<ApplicationDbContext>(options =>
  {
      options.UseNpgsql(connectionString)
             .AddInterceptors(new AuditLogInterceptor(httpContextAccessor));
  });
  ```
