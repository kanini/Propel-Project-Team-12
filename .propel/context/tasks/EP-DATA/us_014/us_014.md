# User Story - US_014

## Story ID
* ID: US_014

## Story Title
* Core User Entity Models (Patient, Staff, Admin)

## Description
* As a backend developer, I want EF Core entity models for Patient, Staff, and Admin with unique email identifiers and role-based attributes, so that the system can differentiate user types and enforce authentication and authorization rules.

## Acceptance Criteria
1. **Given** the Domain layer exists, **When** I create entity classes, **Then** I have three classes:
   - `Patient.cs` with properties: PatientID (GUID primary key), Email (unique string), FirstName, LastName, DateOfBirth, Phone, PasswordHash, Role (enum: Patient), AccountStatus (enum: Active/Inactive), CreatedAt, UpdatedAt
   - `Staff.cs` with properties: StaffID (GUID primary key), Email (unique string), FirstName, LastName, Role (enum: Staff), Specialization (enum: FrontDesk/CallCenter/ClinicalReview), EmployeeID, HireDate, AccountStatus
   - `Admin.cs` with properties: AdminID (GUID primary key), Email (unique string), FirstName, LastName, Role (enum: Admin), Privileges (flags enum: UserManagement/SystemConfig/AuditLogAccess)

2. **Given** the entity models are created, **When** I configure them using Fluent API in `ApplicationDbContext.OnModelCreating`, **Then** each entity has:
   - Email column with unique index: `.HasIndex(e => e.Email).IsUnique()`
   - Required fields: `.Property(e => e.FirstName).IsRequired().HasMaxLength(100)`
   - GUID primary key auto-generated: `.Property(e => e.PatientID).ValueGeneratedOnAdd()`
   - Table names mapped correctly: `.ToTable("Patients")`, `.ToTable("Staff")`, `.ToTable("Admins")`

3. **Given** the entity configurations are complete, **When** I generate a database migration `dotnet ef migrations add AddCoreUserEntities`, **Then** the migration file includes:
   - CREATE TABLE statements for `Patients`, `Staff`, `Admins` with all columns
   - UNIQUE INDEX on `Email` column for each table
   - CHECK constraints for enums (if using PostgreSQL enum types)

4. **Given** the migration is applied, **When** IAttempt to insert duplicate emails via EF Core, **Then** the system throws `DbUpdateException` with constraint violation details.

## Edge Cases
* What happens when email format is invalid? (Email validation enforced at API layer via FluentValidation before reaching database)
* How does the system handle case sensitivity in email uniqueness? (Database collation set to case-insensitive; `user@example.com` equals `USER@EXAMPLE.COM`)

## Technical Notes
- Entity class template:
  ```csharp
  public class Patient
  {
      public Guid PatientID { get; set; }
      public string Email { get; set; } = string.Empty;
      public string FirstName { get; set; } = string.Empty;
      public string LastName { get; set; } = string.Empty;
      public DateTime DateOfBirth { get; set; }
      public string Phone { get; set; } = string.Empty;
      public string PasswordHash { get; set; } = string.Empty;
      public UserRole Role { get; set; } = UserRole.Patient;
      public AccountStatus AccountStatus { get; set; } = AccountStatus.Active;
      public DateTime CreatedAt { get; set; }
      public DateTime UpdatedAt { get; set; }
      
      // Navigation properties
      public ICollection<Appointment> Appointments { get; set; } = new List<Appointment>();
      public ICollection<ClinicalDocument> ClinicalDocuments { get; set; } = new List<ClinicalDocument>();
      public IntakeForm? IntakeForm { get; set; }
  }

  public enum UserRole { Patient, Staff, Admin }
  public enum AccountStatus { Active, Inactive, Suspended }
  ```
- Fluent API configuration in `ApplicationDbContext`:
  ```csharp
  protected override void OnModelCreating(ModelBuilder modelBuilder)
  {
      modelBuilder.Entity<Patient>(entity =>
      {
          entity.HasKey(e => e.PatientID);
          entity.Property(e => e.PatientID).ValueGeneratedOnAdd();
          entity.HasIndex(e => e.Email).IsUnique();
          entity.Property(e => e.Email).IsRequired().HasMaxLength(255);
          entity.Property(e => e.FirstName).IsRequired().HasMaxLength(100);
          entity.Property(e => e.LastName).IsRequired().HasMaxLength(100);
          entity.ToTable("Patients");
      });
      
      // Similar configuration for Staff and Admin
  }
  ```
- Use PostgreSQL constraint names for clarity: `.HasIndex(e => e.Email, "IX_Patients_Email_Unique").IsUnique()`
