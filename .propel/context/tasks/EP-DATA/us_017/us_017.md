# User Story - US_017

## Story ID
* ID: US_017

## Story Title
* Intake and Reminder Entity Models

## Description
* As a backend developer, I want IntakeForm and Reminder entities to store patient questionnaire responses and scheduled notification records, so that the system can track intake completion, preserve edit history, and manage reminder workflows for no-show prevention.

## Acceptance Criteria
1. **Given** the Domain layer exists, **When** I create entity classes, **Then** I have:
   - `IntakeForm.cs` with properties: IntakeFormID (GUID), PatientID (FK), AppointmentID (FK), IntakeMethod (enum: AIConversational/ManualForm), CompletedAt (nullable), LastEditedAt, Demographics (JSON string), MedicalHistory (JSON string), CurrentMedications (JSON string), Allergies (JSON string), ReasonForVisit (text), EditHistory (JSON array with timestamps)
   - `Reminder.cs` with properties: ReminderID (GUID), AppointmentID (FK), ReminderType (enum: Email/SMS), ScheduledFor (DateTime), SentAt (nullable), DeliveryStatus (enum: Pending/Sent/Failed/Bounced), ConfirmationReceived (bool), ConfirmationMethod (enum: EmailClick/SMSReply), ConfirmedAt (nullable)

2. **Given** IntakeForm is configured with Fluent API, **When** I inspect the schema, **Then** I see:
   - Foreign key to Appointment (1:1 relationship): `.HasOne(i => i.Appointment).WithOne(a => a.IntakeForm).HasForeignKey<IntakeForm>(i => i.AppointmentID)`
   - JSON columns for structured data: `.Property(e => e.Demographics).HasColumnType("jsonb")`
   - Nullable CompletedAt to distinguish drafts from completed forms

3. **Given** Reminder is configured, **When** I review relationships, **Then** I see:
   - Foreign key to Appointment (N:1): `.HasOne(r => r.Appointment).WithMany(a => a.Reminders).HasForeignKey(r => r.AppointmentID)`
   - Index on ScheduledFor for efficient background job queries: `.HasIndex(e => new { e.ScheduledFor, e.DeliveryStatus })`
   - Default values: `ConfirmationReceived = false`, `DeliveryStatus = Pending`

4. **Given** the migration is applied, **When** I query draft intake forms, **Then** I can filter by `WHERE CompletedAt IS NULL` to find incomplete forms requiring patient follow-up.

## Edge Cases
* What happens when intake form is edited after 24-hour lock (FR-012)? (Application layer enforces lock; database stores edit attempts in EditHistory with rejection reason)
* How does system handle reminder delivery failures? (DeliveryStatus = Failed; retry logic in application layer with exponential backoff; after 3 failures, manual staff notification)

## Technical Notes
- IntakeForm entity:
  ```csharp
  public class IntakeForm
  {
      public Guid IntakeFormID { get; set; }
      public Guid PatientID { get; set; }
      public Guid AppointmentID { get; set; }
      public IntakeMethod IntakeMethod { get; set; }
      public DateTime? CompletedAt { get; set; } // Null = draft
      public DateTime LastEditedAt { get; set; }
      public string Demographics { get; set; } = "{}"; // JSON
      public string MedicalHistory { get; set; } = "{}"; // JSON
      public string CurrentMedications { get; set; } = "[]"; // JSON array
      public string Allergies { get; set; } = "[]"; // JSON array
      public string ReasonForVisit { get; set; } = string.Empty;
      public string EditHistory { get; set; } = "[]"; // JSON array with timestamps
      
      // Navigation properties
      public Patient Patient { get; set; } = null!;
      public Appointment Appointment { get; set; } = null!;
  }

  public enum IntakeMethod { AIConversational, ManualForm }
  ```
- Reminder entity:
  ```csharp
  public class Reminder
  {
      public Guid ReminderID { get; set; }
      public Guid AppointmentID { get; set; }
      public ReminderType ReminderType { get; set; }
      public DateTime ScheduledFor { get; set; }
      public DateTime? SentAt { get; set; }
      public DeliveryStatus DeliveryStatus { get; set; } = DeliveryStatus.Pending;
      public bool ConfirmationReceived { get; set; } = false;
      public ConfirmationMethod? ConfirmationMethod { get; set; }
      public DateTime? ConfirmedAt { get; set; }
      
      // Navigation properties
      public Appointment Appointment { get; set; } = null!;
  }

  public enum ReminderType { Email, SMS }
  public enum DeliveryStatus { Pending, Sent, Failed, Bounced }
  public enum ConfirmationMethod { EmailClick, SMSReply }
  ```
- Fluent API configuration:
  ```csharp
  modelBuilder.Entity<IntakeForm>(entity =>
  {
      entity.HasOne(e => e.Appointment)
            .WithOne(a => a.IntakeForm)
            .HasForeignKey<IntakeForm>(e => e.AppointmentID);
      entity.Property(e => e.Demographics).HasColumnType("jsonb");
      entity.Property(e => e.MedicalHistory).HasColumnType("jsonb");
      entity.Property(e => e.CurrentMedications).HasColumnType("jsonb");
      entity.Property(e => e.Allergies).HasColumnType("jsonb");
      entity.Property(e => e.EditHistory).HasColumnType("jsonb");
  });

  modelBuilder.Entity<Reminder>(entity =>
  {
      entity.HasIndex(e => new { e.ScheduledFor, e.DeliveryStatus });
      entity.HasOne(e => e.Appointment)
            .WithMany(a => a.Reminders)
            .HasForeignKey(e => e.AppointmentID);
  });
  ```
