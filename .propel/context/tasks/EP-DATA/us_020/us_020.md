# User Story - US_020

## Story ID
* ID: US_020

## Story Title
* Optimistic Concurrency Control Implementation

## Description
* As a backend developer, I want RowVersion optimistic concurrency implemented with exception handling and retry logic, so that concurrent booking conflicts are gracefully resolved with user-friendly conflict resolution messages.

## Acceptance Criteria
1. **Given** the Appointment entity has RowVersion configured (US_015), **When** I implement the booking service, **Then** it includes:
   - Try-catch block wrapping `SaveChangesAsync` to catch `DbUpdateConcurrencyException`
   - Conflict resolution logic that reloads entity from database to check current state
   - Returns HTTP 409 Conflict with message: "Slot no longer available. The appointment was booked by another user. Please select a different time slot."

2. **Given** two concurrent booking requests occur, **When** both transactions execute, **Then**:
   - First transaction commits successfully with new RowVersion value
   - Second transaction detects RowVersion mismatch and throws `DbUpdateConcurrencyException`
   - Exception handler logs conflict details (UserID, SlotDateTime, ConflictReason)
   - API returns alternative available slots with HTTP 409 response

3. **Given** optimistic concurrency is tested, **When** I run integration tests with concurrent requests, **Then** I verify:
   - 100 concurrent booking attempts on same slot result in 1 success and 99 conflicts
   - No duplicate bookings in database (validated by unique constraint on SlotDateTime + ProviderID)
   - All failed transactions rollback cleanly without orphan records

4. **Given** the conflict resolution is UI-friendly, **When** the frontend receives HTTP 409, **Then** it displays:
   - Clear error message banner: "This slot is no longer available"
   - Suggested alternative slots (next 3 available times)
   - "Try Again" button to refresh availability

## Edge Cases
* What happens when RowVersion is null (new entity)? (Initial insert has no concurrency check; RowVersion generated on first save)
* How does system differentiate concurrency conflicts from other update failures? (Catch specific `DbUpdateConcurrencyException` vs generic `DbUpdateException`)

## Technical Notes
- Service layer concurrency handling:
  ```csharp
  public async Task<Result<Appointment>> BookAppointmentAsync(BookAppointmentRequest request)
  {
      var appointment = new Appointment
      {
          AppointmentID = Guid.NewGuid(),
          PatientID = request.PatientID,
          ProviderID = request.ProviderID,
          SlotDateTime = request.SlotDateTime,
          Status = AppointmentStatus.Scheduled,
          BookedAt = DateTime.UtcNow
      };

      try
      {
          _context.Appointments.Add(appointment);
          await _context.SaveChangesAsync();
          return Result.Success(appointment);
      }
      catch (DbUpdateConcurrencyException ex)
      {
          _logger.LogWarning($"Concurrent booking conflict: Slot {request.SlotDateTime} for Provider {request.ProviderID}");

          // Reload entity to get current state
          var entry = ex.Entries.FirstOrDefault();
          if (entry != null)
          {
              await entry.ReloadAsync();
          }

          // Fetch alternative slots
          var alternativeSlots = await GetAlternativeSlots(request.ProviderID, request.SlotDateTime);

          return Result.Conflict(
              "Slot no longer available. Please select another time.",
              alternativeSlots);
      }
      catch (DbUpdateException ex) when (ex.InnerException?.Message.Contains("IX_Appointments_SlotDateTime_ProviderID_Unique") == true)
      {
          // Unique constraint violation (additional safety net)
          _logger.LogError($"Unique constraint violation: {ex.Message}");
          return Result.Conflict("Slot already booked.");
      }
  }
  ```
- API controller response:
  ```csharp
  [HttpPost("book")]
  public async Task<IActionResult> BookAppointment([FromBody] BookAppointmentRequest request)
  {
      var result = await _bookingService.BookAppointmentAsync(request);

      if (result.IsSuccess)
      {
          return Ok(result.Value);
      }

      if (result.StatusCode == 409)
      {
          return Conflict(new
          {
              Message = result.ErrorMessage,
              AlternativeSlots = result.Data
          });
      }

      return StatusCode(result.StatusCode, result.ErrorMessage);
  }
  ```
- Integration test:
  ```csharp
  [Fact]
  public async Task BookAppointment_ConcurrentRequests_OneSuccessRestConflicts()
  {
      // Arrange
      var request = new BookAppointmentRequest
      {
          PatientID = Guid.NewGuid(),
          ProviderID = TestProviderID,
          SlotDateTime = DateTime.UtcNow.AddDays(1)
      };

      // Act: Simulate 100 concurrent requests
      var tasks = Enumerable.Range(0, 100)
          .Select(_ => _bookingService.BookAppointmentAsync(request))
          .ToList();

      var results = await Task.WhenAll(tasks);

      // Assert
      var successCount = results.Count(r => r.IsSuccess);
      var conflictCount = results.Count(r => r.StatusCode == 409);

      Assert.Equal(1, successCount);
      Assert.Equal(99, conflictCount);
  }
  ```
