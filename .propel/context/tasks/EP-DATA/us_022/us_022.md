# User Story - US_022

## Story ID
* ID: US_022

## Story Title
* pgvector Extension Setup

## Description
* As a database administrator, I want the pgvector extension enabled in PostgreSQL with cosine similarity operator configured, so that the system can store 1536-dimension embeddings and perform vector similarity searches for RAG-powered clinical data retrieval.

## Acceptance Criteria
1. **Given** PostgreSQL 14+ is running, **When** I execute the extension creation command, **Then** pgvector is installed:
   - Command: `CREATE EXTENSION IF NOT EXISTS vector;`
   - Verification: `SELECT * FROM pg_extension WHERE extname = 'vector';` returns one row

2. **Given** pgvector is installed, **When** I create a table with vector column, **Then** the column type is accepted:
   - Column definition: `embedding_vector vector(1536)`
   - Sample insert: `INSERT INTO test_table (embedding_vector) VALUES ('[0.1, 0.2, ..., 0.1536]'::vector);`

3. **Given** vector data exists, **When** I perform cosine similarity search, **Then** the `<=>` operator returns distances:
   - Query: `SELECT * FROM ClinicalDocuments ORDER BY EmbeddingVector <=> '[query_vector]'::vector LIMIT 5;`
   - Results ordered by similarity (lower distance = higher similarity)
   - Query uses GiST index from US_019 (verified via EXPLAIN ANALYZE)

4. **Given** the migration is created, **When** I apply it to development and production, **Then** the extension is enabled without errors and supports vector operations.

## Edge Cases
* What happens if pgvector extension is not installed in PostgreSQL instance? (Migration fails with error: "extension 'vector' not found"; requires manual pgvector installation via Docker image or package manager)
* How does system handle dimension mismatch (e.g., inserting 768-dimension vector into 1536-dimension column)? (PostgreSQL raises error: "expected 1536 dimensions, got 768"; application must validate dimensions before insert)

## Technical Notes
- EF Core migration for pgvector extension:
  ```csharp
  migrationBuilder.Sql("CREATE EXTENSION IF NOT EXISTS vector;");
  ```
- Alternative: Run SQL directly in Docker Compose initialization:
  ```yaml
  # docker-compose.yml
  services:
    postgres:
      image: pgvector/pgvector:pg16 # Pre-installed extension
      environment:
        POSTGRES_DB: PatientAccessDb
      command: postgres -c shared_preload_libraries=vector
  ```
- Testing vector operations:
  ```sql
  -- Create test table
  CREATE TABLE test_embeddings (
      id SERIAL PRIMARY KEY,
      content TEXT,
      embedding vector(1536)
  );

  -- Insert sample vector
  INSERT INTO test_embeddings (content, embedding)
  VALUES ('Sample document', array_fill(0.1, ARRAY[1536])::vector);

  -- Cosine similarity search
  SELECT id, content, embedding <=> '[0.2, 0.3, ...]'::vector AS distance
  FROM test_embeddings
  ORDER BY distance
  LIMIT 5;
  ```
- EF Core entity property for vector column:
  ```csharp
  public class ClinicalDocument
  {
      // ... other properties
      public float[]? EmbeddingVector { get; set; } // Mapped to vector(1536)
  }

  // Fluent API configuration
  modelBuilder.Entity<ClinicalDocument>(entity =>
  {
      entity.Property(e => e.EmbeddingVector)
            .HasColumnType("vector(1536)")
            .HasConversion(
                v => v != null ? $"[{string.Join(",", v)}]" : null,
                v => v != null ? v.Trim('[', ']').Split(',').Select(float.Parse).ToArray() : null
            );
  });
  ```
- pgvector similarity functions:
  - `<->`: Euclidean distance (L2)
  - `<#>`: Negative inner product
  - `<=>`: Cosine distance (1 - cosine similarity) — **Recommended for text embeddings**
- Index type selection:
  - **GiST**: Good for smaller datasets (<100K vectors), supports all distance functions
  - **HNSW**: Better for larger datasets (>100K vectors), requires `pgvector` version 0.5.0+
  - **Recommended for this project**: GiST index (sufficient for 10K patient records per NFR-014)
- Create GiST index (done in US_019):
  ```sql
  CREATE INDEX IX_ClinicalDocuments_EmbeddingVector_GiST 
  ON ClinicalDocuments USING gist (EmbeddingVector vector_cosine_ops);
  ```
