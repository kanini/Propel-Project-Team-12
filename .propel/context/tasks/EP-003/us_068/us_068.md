# User Story - US_068

## Story ID

* ID: US_068

## Story Title

* Staff Dashboard - Operations Hub

## Description

* As a staff member, I want a dashboard as my post-login landing page showing today's queue summary, pending verification alerts, walk-in booking quick access, and real-time arrival updates, so that I can efficiently manage daily operations and prioritize urgent tasks.

## Acceptance Criteria

1. **Given** I am a staff member who has just logged in, **When** authentication completes successfully (UC-002), **Then** the system redirects me to the Staff Dashboard (SCR-004) displaying operational metrics within 500ms (NFR-001).

2. **Given** I am viewing the Staff Dashboard, **When** the page renders, **Then** the system displays three statistical cards showing: today's total appointments, current queue size (patients waiting), and pending verifications (unverified clinical data items) with real-time updates.

3. **Given** I am viewing the dashboard, **When** the page loads, **Then** the system displays three quick action cards: "Walk-in Booking" (navigates to SCR-018), "View Queue" (navigates to SCR-019), and "Mark Arrivals" (navigates to SCR-020), meeting UXR-001 (max 3 clicks to any feature).

4. **Given** there are patients in today's queue, **When** the dashboard renders, **Then** the system displays a "Today's Queue" widget showing the next 5 patients chronologically with name, appointment time, provider, and estimated wait time per UC-012.

5. **Given** there are pending clinical data verifications, **When** the dashboard loads, **Then** the system displays a "Pending Verifications" widget showing the 5 most critical unverified items (prioritized by risk level: Critical > High > Medium) with patient name, data type, and "Verify" action link per UXR-402.

6. **Given** I need to view high-risk appointments, **When** the dashboard renders, **Then** the system displays risk indicator badges (red/amber/green) alongside appointments in the queue widget based on no-show risk assessment per FR-023.

7. **Given** a patient is marked as "Arrived" by another staff member, **When** the arrival status changes, **Then** the system updates the dashboard queue widget within 2 seconds via real-time updates (Pusher Channels) per TR-012.

8. **Given** I click any of the quick action cards or queue items, **When** the action executes, **Then** the system provides visual feedback within 200ms and navigates to the target screen per UXR-501.

9. **Given** the dashboard is loading data, **When** API response time exceeds 300ms, **Then** the system displays skeleton loading states for stat cards, queue widget, and verifications panel per UXR-502.

10. **Given** there are no patients scheduled today, **When** the dashboard loads, **Then** the system displays empty state illustration with CTA per UXR-605: "No appointments scheduled today. Use Walk-in Booking to add patients."

## Edge Cases

* What happens when queue data fetch fails? Display error banner with retry action per UXR-603 while showing empty placeholder for queue widget.
* How does the system handle 50+ pending verifications? Display top 5 most critical (by risk score) with "View All Verifications" link to SCR-023.
* What happens when a patient cancels last-minute via patient portal? Real-time update removes appointment from queue and decrements queue count on dashboard within 2 seconds.
* How does the dashboard prioritize urgent vs routine tasks? Visual indicators: Critical verifications with red badge shown first, emergency queue priority marked with warning icon.
