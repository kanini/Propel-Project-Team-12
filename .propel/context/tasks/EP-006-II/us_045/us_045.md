# User Story - US_045

## Story ID

* ID: US_045

## Story Title

* AI Clinical Data Extraction from PDFs

## Description

* As a system, I want to extract structured clinical data (vitals, history, medications, allergies, labs, diagnoses) from uploaded PDF documents using Azure AI Document Intelligence, so that unstructured clinical records are transformed into actionable health data.

## Acceptance Criteria

1. **Given** a PDF document is queued for processing, **When** the extraction job runs, **Then** Azure AI Document Intelligence processes the document and extracts vitals, medical history, medications, allergies, lab results, and diagnoses as structured data points.

2. **Given** data is extracted, **When** each data point is stored, **Then** it includes a confidence score (0-100%) and a source document reference (page number, text excerpt) enabling traceability to the original document.

3. **Given** the extraction is running, **When** processing a single page, **Then** AI inference completes within 5 seconds at the 95th percentile (AIR-Q02, NFR-013).

4. **Given** the AI extraction targets quality, **When** evaluated against a test set, **Then** hallucination rate is below 2% (AIR-Q04) and extraction recall is above 95% for critical elements like medications and allergies (AIR-Q05).

5. **Given** multiple data types exist in a document, **When** extraction completes, **Then** each element is classified by type (Vital, Medication, Allergy, Diagnosis, LabResult) with appropriate structured fields.

## Edge Cases

* What happens when a PDF page is a scanned image with poor quality? AI returns lower confidence scores; data points below 50% confidence are flagged for manual review.
* How does the system handle multi-language content in a document? English content is extracted; non-English sections are skipped with a note in the extraction results.
