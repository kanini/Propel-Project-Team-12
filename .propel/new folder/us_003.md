# User Story - US_003

## Story ID

- ID: US_003

## Story Title

- Session Timeout with Auto-Save and Re-Authentication

## Description

- As an authenticated user, I want the system to warn me before my session expires and automatically save my work, so that I can extend my session or re-authenticate without losing my data.

## Acceptance Criteria

1. **Given** I am authenticated and active in the system, **When** I have been inactive for 13 minutes (2 minutes before the 15-minute timeout), **Then** a modal dialog appears warning "Your session will expire in 2 minutes" with options to "Extend Session" or "Logout".

2. **Given** the session timeout warning modal is displayed, **When** I click "Extend Session", **Then** the session timer resets to 15 minutes and the modal closes, allowing me to continue working.

3. **Given** the session timeout warning modal is displayed, **When** I click "Logout" or ignore the warning, **Then** after 2 more minutes the system logs me out, auto-saves any form data, and redirects to the sign-in page.

4. **Given** I am filling out a form (intake, booking confirmation, etc.), **When** my session expires, **Then** the system auto-saves all entered form data to local storage/session storage before redirecting to sign-in.

5. **Given** I have been logged out due to session timeout, **When** I successfully re-authenticate, **Then** the system restores my previous context including any auto-saved form data, and displays a notification "Your previous session data has been restored."

6. **Given** I am authenticated and interacting with the system, **When** I perform any action (click, type, scroll), **Then** the inactivity timer resets, preventing premature timeout warnings.

7. **Given** the timeout warning modal is visible, **When** I perform any action outside the modal, **Then** the modal remains visible (non-dismissable by clicking outside) to ensure user attention.

## Edge Cases

- What happens if auto-save fails (e.g., storage full)? (System attempts to display warning; logs error for support)
- How does the system handle timeout during an API call in progress? (API call completes; response handled after re-authentication)
- What happens if user has multiple browser tabs open? (Session extends in all tabs when any tab shows activity)
- How does the system handle network disconnection during timeout? (Timer continues locally; offline indicator shown per UXR-605)
